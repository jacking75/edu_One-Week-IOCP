# Chapter 3. 클라이언트 연결 관리
실제 게임 서버에서는 수백, 수천 명의 플레이어가 동시에 접속하고 게임을 플레이한다. Chapter 2에서 구현한 Echo 서버는 기본적인 IOCP의 동작 원리를 이해하기에는 충분했지만, 실제 게임 서버 환경에서 요구되는 대규모 동시 접속자 처리에는 미흡한 점이 많다. 이 챕터에서는 클라이언트의 연결을 효율적이고 안전하게 관리하는 방법을 학습한다.

클라이언트 연결 관리는 게임 서버의 가장 기본이 되는 부분이면서도 가장 중요한 부분이다. 연결이 제대로 관리되지 않으면 메모리 누수, 핸들 누수, 좀비 세션 등의 문제가 발생하여 서버의 안정성과 성능이 크게 저하된다. 이번 챕터에서는 고성능 Accept 처리, 세션 객체 설계, 세션 풀 관리, 그리고 안전한 연결 해제 처리까지 실전에서 필요한 모든 기법을 다룬다.

## 3.1 동기 Accept vs 비동기 Accept

**동기 Accept의 한계**  
Chapter 2에서 구현한 Echo 서버에서는 별도의 Accept 스레드를 만들고, 그 스레드에서 `accept()` 함수를 호출하여 클라이언트 연결을 받아들였다. 이 방식은 구현이 간단하고 이해하기 쉽다는 장점이 있지만, 성능과 확장성 측면에서 몇 가지 문제점을 가지고 있다.

```cpp
// 동기 Accept 방식 (Chapter 2에서 사용한 방식)
SOCKET clientSocket = accept(listenSocket, (SOCKADDR*)&clientAddr, &addrLen);
if (clientSocket == INVALID_SOCKET) {
    // 에러 처리
    return;
}

// IOCP에 소켓 연결
CreateIoCompletionPort((HANDLE)clientSocket, iocpHandle, 
                       (ULONG_PTR)session, 0);
```

동기 Accept 방식의 문제점은 다음과 같다. 첫째, `accept()` 함수는 클라이언트 연결이 들어올 때까지 블로킹되어 대기한다. 연결 요청이 없으면 스레드는 계속 대기 상태로 머물게 되어 리소스를 낭비하게 된다. 둘째, 동시에 여러 클라이언트가 접속을 시도할 경우, Accept 스레드가 하나씩 순차적으로 처리해야 하므로 병목이 발생할 수 있다. 특히 게임 서버가 오픈하는 순간이나 점검 후 재접속 시점처럼 짧은 시간에 대량의 연결이 몰리는 상황에서는 이러한 문제가 심각해진다.

셋째, `accept()` 호출 후에 `CreateIoCompletionPort()`를 호출하여 소켓을 IOCP에 연결하는 작업이 추가로 필요하다. 이 과정에서 컨텍스트 스위칭과 커널 모드 전환이 발생하여 오버헤드가 증가한다. 넷째, Accept 스레드와 워커 스레드가 분리되어 있어서 세션 객체의 생명주기 관리가 복잡해진다.

**비동기 Accept의 필요성**  
IOCP는 비동기 I/O를 위한 모델이다. 송수신뿐만 아니라 Accept 작업도 비동기로 처리할 수 있다면 IOCP의 장점을 최대한 활용할 수 있다. Windows는 `AcceptEx`라는 확장 함수를 제공하여 비동기 Accept를 지원한다. `AcceptEx`를 사용하면 Accept 작업도 Overlapped I/O로 처리되어 완료 통지가 IOCP를 통해 전달된다.

비동기 Accept 방식의 장점은 다음과 같다. Accept 전용 스레드가 필요 없어진다. 워커 스레드들이 Accept 완료 통지도 함께 처리하므로 스레드 관리가 단순해진다. 동시에 여러 개의 Accept 작업을 대기시킬 수 있어서 대량 접속 상황에서도 빠르게 처리할 수 있다. Accept와 동시에 첫 번째 데이터까지 받을 수 있어서 네트워크 왕복을 줄일 수 있다. IOCP 워커 스레드 풀을 활용하므로 시스템 리소스를 효율적으로 사용한다.

```
[동기 Accept 흐름]
Client Connect → Listen Socket → Accept Thread (blocking)
                                       ↓
                                  Create Session
                                       ↓
                                Bind to IOCP
                                       ↓
                                Start Async Recv

[비동기 Accept 흐름]
Client Connect → Listen Socket → AcceptEx (queued)
                                       ↓
                                IOCP Completion
                                       ↓
                      Worker Thread processes connection
                                       ↓
                                Start Async Recv
```

## 3.2 AcceptEx를 이용한 고성능 Accept

**AcceptEx 함수 이해하기**  
`AcceptEx`는 Winsock의 확장 함수로, `<mswsock.h>` 헤더에 선언되어 있다. 이 함수는 일반적인 Winsock 함수처럼 직접 링크되는 것이 아니라, `WSAIoctl` 함수를 통해 함수 포인터를 얻어와야 한다. 이는 운영체제 버전이나 서비스 프로바이더에 따라 최적화된 구현을 제공하기 위함이다.

```cpp
// AcceptEx 함수 포인터 타입
typedef BOOL (WINAPI *LPFN_ACCEPTEX)(
    SOCKET sListenSocket,
    SOCKET sAcceptSocket,
    PVOID lpOutputBuffer,
    DWORD dwReceiveDataLength,
    DWORD dwLocalAddressLength,
    DWORD dwRemoteAddressLength,
    LPDWORD lpdwBytesReceived,
    LPOVERLAPPED lpOverlapped
);

// AcceptEx 함수 포인터 얻기
LPFN_ACCEPTEX AcceptEx = nullptr;
GUID guidAcceptEx = WSAID_ACCEPTEX;
DWORD bytes = 0;

WSAIoctl(listenSocket, 
         SIO_GET_EXTENSION_FUNCTION_POINTER,
         &guidAcceptEx, sizeof(guidAcceptEx),
         &AcceptEx, sizeof(AcceptEx),
         &bytes, nullptr, nullptr);
```

`AcceptEx` 함수의 매개변수를 살펴보자. `sListenSocket`은 listen 상태의 소켓이고, `sAcceptSocket`은 미리 생성해둔 연결용 소켓이다. 이것이 동기 `accept()`와의 큰 차이점인데, `accept()`는 내부적으로 소켓을 생성해서 반환하지만, `AcceptEx`는 호출자가 미리 소켓을 생성해서 전달해야 한다.

`lpOutputBuffer`는 클라이언트의 주소 정보와 첫 번째 수신 데이터를 받을 버퍼다. `dwReceiveDataLength`는 실제 데이터를 받을 공간의 크기이고, `dwLocalAddressLength`와 `dwRemoteAddressLength`는 로컬 주소와 원격 주소를 저장할 공간의 크기다. 일반적으로 `sizeof(SOCKADDR_STORAGE) + 16` 바이트를 지정한다. 16바이트는 내부적으로 사용되는 여유 공간이다.

**AcceptEx 사용 패턴**  
`AcceptEx`를 사용하는 기본적인 패턴은 다음과 같다. 먼저 Listen 소켓을 IOCP에 연결한다. 그리고 미리 여러 개의 Accept 작업을 대기시켜 놓는다. 이를 "Pre-posted Accept"라고 부른다. 일반적으로 CPU 코어 수의 2배 정도를 대기시키는 것이 좋다.

```cpp
// Accept용 Overlapped 구조체 확장
struct AcceptOverlapped {
    OVERLAPPED overlapped;
    SOCKET acceptSocket;
    BYTE buffer[64]; // 주소 정보를 받을 버퍼
};

// Listen 소켓을 IOCP에 연결
CreateIoCompletionPort((HANDLE)listenSocket, iocpHandle, 
                       (ULONG_PTR)KEY_ACCEPT, 0);

// 여러 개의 Accept 작업 대기
for (int i = 0; i < prePostedAcceptCount; ++i) {
    PostAccept();
}
```

`PostAccept` 함수는 하나의 비동기 Accept 작업을 시작한다. 먼저 연결을 받을 새로운 소켓을 생성하고, `AcceptEx`를 호출한다.

```cpp
void PostAccept() {
    AcceptOverlapped* acceptOv = new AcceptOverlapped{};
    ZeroMemory(&acceptOv->overlapped, sizeof(OVERLAPPED));
    
    acceptOv->acceptSocket = WSASocket(AF_INET, SOCK_STREAM, IPPROTO_TCP,
                                       nullptr, 0, WSA_FLAG_OVERLAPPED);
    
    DWORD bytes = 0;
    BOOL result = AcceptEx(
        listenSocket,
        acceptOv->acceptSocket,
        acceptOv->buffer,
        0,  // 첫 데이터를 받지 않음
        sizeof(SOCKADDR_STORAGE) + 16,
        sizeof(SOCKADDR_STORAGE) + 16,
        &bytes,
        &acceptOv->overlapped
    );
    
    if (!result && WSAGetLastError() != ERROR_IO_PENDING) {
        // 에러 처리
        closesocket(acceptOv->acceptSocket);
        delete acceptOv;
    }
}
```

워커 스레드에서 Accept 완료 통지를 받으면, 연결된 클라이언트를 위한 세션을 생성하고 초기화한다. 중요한 점은 Accept가 완료된 후 반드시 `setsockopt`로 `SO_UPDATE_ACCEPT_CONTEXT` 옵션을 설정해야 한다는 것이다. 이 옵션을 설정하지 않으면 `getpeername`, `getsockname` 등의 함수가 제대로 동작하지 않는다.

```cpp
// 워커 스레드에서 Accept 완료 처리
if (key == KEY_ACCEPT) {
    AcceptOverlapped* acceptOv = (AcceptOverlapped*)overlapped;
    
    // Accept 컨텍스트 업데이트
    setsockopt(acceptOv->acceptSocket, SOL_SOCKET, 
               SO_UPDATE_ACCEPT_CONTEXT,
               (char*)&listenSocket, sizeof(listenSocket));
    
    // 세션 생성 및 초기화
    Session* session = CreateSession(acceptOv->acceptSocket);
    
    // 새로운 Accept 대기
    PostAccept();
    
    delete acceptOv;
}
```

**GetAcceptExSockaddrs로 주소 정보 추출**  
`AcceptEx`의 출력 버퍼에는 로컬 주소와 원격 주소 정보가 특별한 형식으로 저장된다. 이 정보를 추출하기 위해서는 `GetAcceptExSockaddrs` 함수를 사용해야 한다. 이 함수도 `AcceptEx`처럼 `WSAIoctl`을 통해 함수 포인터를 얻어와야 한다.

```cpp
// GetAcceptExSockaddrs 함수 포인터 얻기
LPFN_GETACCEPTEXSOCKADDRS GetAcceptExSockaddrs = nullptr;
GUID guidGetAcceptExSockaddrs = WSAID_GETACCEPTEXSOCKADDRS;

WSAIoctl(listenSocket,
         SIO_GET_EXTENSION_FUNCTION_POINTER,
         &guidGetAcceptExSockaddrs, sizeof(guidGetAcceptExSockaddrs),
         &GetAcceptExSockaddrs, sizeof(GetAcceptExSockaddrs),
         &bytes, nullptr, nullptr);

// 주소 정보 추출
SOCKADDR* localAddr = nullptr;
SOCKADDR* remoteAddr = nullptr;
int localAddrLen = 0;
int remoteAddrLen = 0;

GetAcceptExSockaddrs(
    acceptOv->buffer,
    0,  // 데이터 길이
    sizeof(SOCKADDR_STORAGE) + 16,
    sizeof(SOCKADDR_STORAGE) + 16,
    &localAddr, &localAddrLen,
    &remoteAddr, &remoteAddrLen
);

// IP 주소 출력 (디버깅용)
char clientIP[INET_ADDRSTRLEN];
inet_ntop(AF_INET, &((SOCKADDR_IN*)remoteAddr)->sin_addr,
          clientIP, sizeof(clientIP));
std::cout << "새 연결: " << clientIP << std::endl;
```

## 3.3 세션(Session) 객체 설계

**세션 객체의 역할**  
게임 서버에서 세션(Session)은 연결된 클라이언트 하나를 나타내는 객체다. 소켓, 송수신 버퍼, 플레이어 정보, 게임 상태 등 클라이언트와 관련된 모든 데이터를 하나의 객체로 캡슐화한다. 세션 객체를 잘 설계하면 코드의 가독성과 유지보수성이 크게 향상된다.

세션 객체가 가져야 할 기본적인 책임은 다음과 같다. 소켓 관리 및 생명주기 제어, 비동기 송수신 작업 관리, 연결 상태 추적, 송수신 버퍼 관리, 클라이언트 정보 저장이다. 이 챕터에서는 버퍼 관리를 제외한 기본적인 세션 객체를 설계하고, Chapter 4에서 링 버퍼를 추가할 것이다.

**세션 상태 관리**  
세션은 여러 가지 상태를 가질 수 있다. 연결 중(Connecting), 연결됨(Connected), 연결 해제 중(Disconnecting), 연결 해제됨(Disconnected) 등의 상태가 있다. 상태를 명확히 관리하지 않으면 이미 끊긴 세션에 데이터를 보내거나, 이미 해제된 메모리에 접근하는 등의 문제가 발생할 수 있다.

```cpp
enum class SessionState {
    Connecting,    // AcceptEx 완료, 초기화 중
    Connected,     // 정상 연결, 사용 가능
    Disconnecting, // 연결 해제 시작
    Disconnected   // 완전히 해제됨
};

class Session {
private:
    SOCKET socket_;
    SessionState state_;
    std::atomic<int32_t> refCount_;  // 참조 카운트
    uint64_t sessionId_;             // 고유 ID
    
    // Overlapped 구조체 확장
    struct RecvOverlapped {
        OVERLAPPED overlapped;
        Session* owner;
    };
    
    struct SendOverlapped {
        OVERLAPPED overlapped;
        Session* owner;
        std::vector<BYTE> buffer;
    };
    
    RecvOverlapped recvOverlapped_;
    BYTE recvBuffer_[4096];
    
public:
    Session(SOCKET socket, uint64_t sessionId);
    ~Session();
    
    void Initialize();
    void Disconnect();
    void ProcessRecv(DWORD bytesTransferred);
    void ProcessSend(DWORD bytesTransferred);
    
    void PostRecv();
    void PostSend(const BYTE* data, int32_t len);
    
    void AddRef();
    void Release();
};
```

세션 객체는 참조 카운팅을 사용하여 생명주기를 관리한다. IOCP 환경에서는 여러 워커 스레드가 동시에 하나의 세션에 접근할 수 있으므로, 안전하게 해제하기 위해서는 참조 카운팅이 필수적이다. `std::atomic<int32_t>`를 사용하여 스레드 안전한 참조 카운트를 구현한다.

```cpp
Session::Session(SOCKET socket, uint64_t sessionId)
    : socket_(socket)
    , state_(SessionState::Connecting)
    , refCount_(1)  // 초기 참조 카운트는 1
    , sessionId_(sessionId)
{
    ZeroMemory(&recvOverlapped_, sizeof(recvOverlapped_));
    recvOverlapped_.owner = this;
}

void Session::AddRef() {
    refCount_.fetch_add(1);
}

void Session::Release() {
    int32_t refCount = refCount_.fetch_sub(1) - 1;
    if (refCount == 0) {
        delete this;
    }
}
```

**세션 초기화와 비동기 수신 시작**  
세션이 생성되면 IOCP에 소켓을 연결하고, 첫 번째 비동기 수신을 시작한다. 소켓을 IOCP에 연결할 때 `CompletionKey`로 세션 객체의 포인터를 전달하면, 나중에 완료 통지를 받을 때 어떤 세션에 대한 작업인지 바로 알 수 있다.

```cpp
void Session::Initialize() {
    // IOCP에 소켓 연결
    CreateIoCompletionPort((HANDLE)socket_, g_iocpHandle,
                           (ULONG_PTR)this, 0);
    
    // 상태 변경
    state_ = SessionState::Connected;
    
    // 첫 번째 비동기 수신 시작
    PostRecv();
}

void Session::PostRecv() {
    if (state_ != SessionState::Connected)
        return;
    
    ZeroMemory(&recvOverlapped_.overlapped, sizeof(OVERLAPPED));
    
    WSABUF wsaBuf;
    wsaBuf.buf = (char*)recvBuffer_;
    wsaBuf.len = sizeof(recvBuffer_);
    
    DWORD flags = 0;
    DWORD bytesReceived = 0;
    
    // 참조 카운트 증가 (비동기 작업이 완료될 때까지 세션 유지)
    AddRef();
    
    int result = WSARecv(socket_, &wsaBuf, 1, &bytesReceived,
                         &flags, &recvOverlapped_.overlapped, nullptr);
    
    if (result == SOCKET_ERROR) {
        int error = WSAGetLastError();
        if (error != WSA_IO_PENDING) {
            Release();  // 실패 시 참조 카운트 감소
            Disconnect();
        }
    }
}
```

**연결 해제 처리**  
연결 해제는 단순히 소켓을 닫는 것 이상의 작업이 필요하다. 진행 중인 비동기 작업이 완료될 때까지 기다려야 하며, 세션 객체가 사용 중인 리소스를 정리해야 한다. `Disconnect` 함수는 여러 스레드에서 동시에 호출될 수 있으므로, 중복 실행을 방지하는 메커니즘이 필요하다.

```cpp
void Session::Disconnect() {
    SessionState expected = SessionState::Connected;
    
    // CAS(Compare-And-Swap)를 사용하여 한 번만 실행되도록 보장
    if (!std::atomic_compare_exchange_strong(
        reinterpret_cast<std::atomic<SessionState>*>(&state_),
        &expected, SessionState::Disconnecting)) {
        return;  // 이미 다른 스레드가 연결 해제 중
    }
    
    // 소켓 종료
    closesocket(socket_);
    socket_ = INVALID_SOCKET;
    
    // 상태 변경
    state_ = SessionState::Disconnected;
    
    // 세션 매니저에 통지
    g_sessionManager->RemoveSession(sessionId_);
    
    // 참조 카운트 감소 (생성 시 초기 참조)
    Release();
}
```

## 3.4 세션 풀(Pool) 관리 기법

**세션 풀의 필요성**  
게임 서버는 플레이어가 접속하고 나가는 일이 매우 빈번하게 발생한다. 접속할 때마다 세션 객체를 동적 할당하고, 접속 해제 시 삭제하는 방식은 메모리 할당/해제 오버헤드가 크다. 특히 `new`와 `delete`는 힙 락(Heap Lock)을 획득해야 하므로, 멀티스레드 환경에서는 성능 병목이 될 수 있다.

세션 풀(Session Pool)은 미리 일정 개수의 세션 객체를 생성해두고, 필요할 때 꺼내 쓰고 반납하는 방식이다. 이를 통해 메모리 할당 오버헤드를 제거하고, 메모리 단편화를 방지할 수 있다. 또한 최대 동시 접속자 수를 제한하는 효과도 있어서 서버 리소스를 보호할 수 있다.

```
[세션 풀 구조]

┌────────────────────────────────────┐
│     Session Pool Manager           │
├────────────────────────────────────┤
│  Available Sessions (Queue)        │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐      │
│  │ S1 │→│ S2 │→│ S3 │→│ S4 │→...  │
│  └────┘ └────┘ └────┘ └────┘      │
├────────────────────────────────────┤
│  Active Sessions (Map)             │
│  SessionID → Session*              │
│  1001 → S5                         │
│  1002 → S6                         │
│  1003 → S7                         │
└────────────────────────────────────┘
```

**Lock-Free 세션 풀 구현**  
세션 풀은 여러 워커 스레드에서 동시에 접근하므로 스레드 안전성이 매우 중요하다. 간단한 방법은 `std::mutex`로 보호하는 것이지만, 더 나은 성능을 위해 Lock-Free 자료구조를 사용할 수 있다. C++20의 `std::atomic`을 활용하면 비교적 간단하게 Lock-Free 스택을 구현할 수 있다.

```cpp
class SessionPool {
private:
    struct Node {
        Session* session;
        Node* next;
    };
    
    std::atomic<Node*> freeList_;  // Lock-Free Stack
    std::unordered_map<uint64_t, Session*> activeSessions_;
    std::mutex activeSessionsMutex_;  // 활성 세션 맵은 뮤텍스 사용
    
    std::atomic<uint64_t> sessionIdGenerator_;
    const int32_t maxSessions_;
    
public:
    SessionPool(int32_t maxSessions)
        : freeList_(nullptr)
        , sessionIdGenerator_(1)
        , maxSessions_(maxSessions)
    {
        // 세션 풀 초기화
        for (int32_t i = 0; i < maxSessions_; ++i) {
            Node* node = new Node;
            node->session = new Session(INVALID_SOCKET, 0);
            node->next = freeList_.load();
            
            while (!freeList_.compare_exchange_weak(node->next, node)) {
                // CAS 재시도
            }
        }
    }
    
    Session* Acquire(SOCKET socket) {
        // Lock-Free Stack에서 Pop
        Node* node = freeList_.load();
        while (node != nullptr) {
            Node* next = node->next;
            if (freeList_.compare_exchange_weak(node, next)) {
                break;
            }
        }
        
        if (node == nullptr) {
            return nullptr;  // 풀이 비었음
        }
        
        // 세션 초기화
        uint64_t sessionId = sessionIdGenerator_.fetch_add(1);
        Session* session = node->session;
        session->Reset(socket, sessionId);
        
        delete node;
        
        // 활성 세션 맵에 추가
        {
            std::lock_guard<std::mutex> lock(activeSessionsMutex_);
            activeSessions_[sessionId] = session;
        }
        
        return session;
    }
    
    void Release(Session* session) {
        uint64_t sessionId = session->GetSessionId();
        
        // 활성 세션 맵에서 제거
        {
            std::lock_guard<std::mutex> lock(activeSessionsMutex_);
            activeSessions_.erase(sessionId);
        }
        
        // Lock-Free Stack에 Push
        Node* node = new Node;
        node->session = session;
        node->next = freeList_.load();
        
        while (!freeList_.compare_exchange_weak(node->next, node)) {
            // CAS 재시도
        }
    }
};
```

실전에서는 완전한 Lock-Free 구현보다는 `std::mutex`를 사용한 단순한 구현이 더 나을 수도 있다. Lock-Free 자료구조는 구현이 복잡하고 디버깅이 어려우며, 실제 성능 향상이 미미한 경우도 있다. 프로파일링을 통해 병목이 확인된 경우에만 Lock-Free 구현을 고려하는 것이 좋다.

**세션 풀을 사용한 Accept 처리**  
세션 풀을 사용하면 Accept 완료 처리가 간결해진다. 풀에서 세션을 가져와서 초기화하고, IOCP에 연결하면 된다.

```cpp
// 워커 스레드에서 Accept 완료 처리
if (key == KEY_ACCEPT) {
    AcceptOverlapped* acceptOv = (AcceptOverlapped*)overlapped;
    
    // Accept 컨텍스트 업데이트
    setsockopt(acceptOv->acceptSocket, SOL_SOCKET,
               SO_UPDATE_ACCEPT_CONTEXT,
               (char*)&listenSocket, sizeof(listenSocket));
    
    // 세션 풀에서 세션 획득
    Session* session = g_sessionPool->Acquire(acceptOv->acceptSocket);
    
    if (session != nullptr) {
        session->Initialize();
        std::cout << "세션 연결됨: " << session->GetSessionId() << std::endl;
    } else {
        // 풀이 비었으면 연결 거부
        std::cout << "세션 풀 고갈, 연결 거부" << std::endl;
        closesocket(acceptOv->acceptSocket);
    }
    
    // 새로운 Accept 대기
    PostAccept();
    
    delete acceptOv;
}
```

## 3.5 연결 해제 처리 (Graceful Shutdown)

**정상 종료 vs 비정상 종료**  
TCP 연결을 해제하는 방법에는 여러 가지가 있다. 가장 이상적인 방법은 양측이 모두 동의하여 정상적으로 종료하는 Graceful Shutdown이다. 이 방식은 `shutdown()` 함수를 사용하여 연결의 송신 방향을 먼저 닫고, 상대방이 FIN을 보내올 때까지 기다린 후 소켓을 닫는다.

반면 비정상 종료는 즉시 연결을 끊는 방법이다. `closesocket()`을 바로 호출하거나, `SO_LINGER` 옵션을 0으로 설정하여 RST 패킷을 보내는 방식이다. 이 방법은 빠르지만, 송신 버퍼에 남아있는 데이터가 유실될 수 있고, 상대방이 TIME_WAIT 상태에 빠질 수 있다.

```cpp
// Graceful Shutdown
shutdown(socket, SD_SEND);  // 송신 방향 종료
// 상대방의 FIN을 기다림
closesocket(socket);

// Hard Close (즉시 종료)
LINGER linger;
linger.l_onoff = 1;
linger.l_linger = 0;
setsockopt(socket, SOL_SOCKET, SO_LINGER, 
           (char*)&linger, sizeof(linger));
closesocket(socket);
```

게임 서버에서는 상황에 따라 적절한 방법을 선택해야 한다. 정상적인 로그아웃이나 게임 종료 시에는 Graceful Shutdown을 사용하여 마지막 패킷까지 안전하게 전송하는 것이 좋다. 반면 타임아웃이나 프로토콜 위반 등의 비정상 상황에서는 즉시 연결을 끊는 것이 합리적이다.

**DisconnectEx를 이용한 비동기 연결 해제**   
`AcceptEx`처럼 연결 해제도 비동기로 처리할 수 있다. `DisconnectEx` 함수를 사용하면 연결 해제 작업이 Overlapped I/O로 처리되어, 완료 통지를 IOCP를 통해 받을 수 있다. 이를 통해 연결 해제 후 세션을 안전하게 정리하고, 소켓을 재사용할 수도 있다.

```cpp
// DisconnectEx 함수 포인터 얻기
LPFN_DISCONNECTEX DisconnectEx = nullptr;
GUID guidDisconnectEx = WSAID_DISCONNECTEX;
DWORD bytes = 0;

WSAIoctl(socket,
         SIO_GET_EXTENSION_FUNCTION_POINTER,
         &guidDisconnectEx, sizeof(guidDisconnectEx),
         &DisconnectEx, sizeof(DisconnectEx),
         &bytes, nullptr, nullptr);

// 비동기 연결 해제
struct DisconnectOverlapped {
    OVERLAPPED overlapped;
    Session* owner;
};

DisconnectOverlapped* disconnectOv = new DisconnectOverlapped{};
disconnectOv->owner = this;

BOOL result = DisconnectEx(socket_, &disconnectOv->overlapped,
                           TF_REUSE_SOCKET, 0);

if (!result && WSAGetLastError() != ERROR_IO_PENDING) {
    // 에러 처리
    delete disconnectOv;
    closesocket(socket_);
}
```

`TF_REUSE_SOCKET` 플래그를 사용하면 `DisconnectEx` 완료 후 소켓을 즉시 재사용할 수 있다. 이는 세션 풀에서 소켓까지 재사용하고자 할 때 유용하다. 하지만 대부분의 경우 세션 객체만 재사용하고 소켓은 매번 새로 생성하는 것이 더 안전하고 단순하다.

**송신 버퍼 비우기**  
연결을 해제하기 전에 송신 버퍼에 남아있는 데이터를 모두 전송해야 하는 경우가 있다. 예를 들어 "서버 점검으로 인한 접속 종료" 메시지를 보낸 후 연결을 끊어야 할 때다. 이런 경우 송신이 완료된 후에 연결을 해제해야 한다.

```cpp
void Session::DisconnectAfterSend() {
    pendingDisconnect_ = true;
    
    // 송신 중인 작업이 없으면 즉시 연결 해제
    if (sendInProgress_.load() == 0) {
        Disconnect();
    }
    // 그렇지 않으면 송신 완료 시 Disconnect 호출
}

void Session::ProcessSend(DWORD bytesTransferred) {
    // 송신 처리...
    
    int32_t sendCount = sendInProgress_.fetch_sub(1) - 1;
    
    // 송신 완료 후 연결 해제 대기 중이면
    if (pendingDisconnect_ && sendCount == 0) {
        Disconnect();
    }
}
```

**타임아웃 처리**  
클라이언트가 비정상 종료되거나 네트워크가 끊긴 경우, 서버 측에서는 이를 감지하기 어렵다. TCP의 Keepalive 메커니즘이 있지만, 기본 타임아웃이 2시간으로 매우 길어서 실시간 게임에는 적합하지 않다. 따라서 애플리케이션 레벨에서 자체적으로 타임아웃을 구현해야 한다.

간단한 방법은 마지막 수신 시간을 기록해두고, 주기적으로 현재 시간과 비교하여 일정 시간 이상 통신이 없으면 연결을 끊는 것이다. Chapter 9에서 타이머 시스템을 구현할 때 더 체계적인 방법을 다룰 것이다.

```cpp
class Session {
private:
    std::chrono::steady_clock::time_point lastRecvTime_;
    
public:
    void UpdateRecvTime() {
        lastRecvTime_ = std::chrono::steady_clock::now();
    }
    
    bool IsTimeout(int32_t timeoutSeconds) const {
        auto now = std::chrono::steady_clock::now();
        auto elapsed = std::chrono::duration_cast<std::chrono::seconds>(
            now - lastRecvTime_).count();
        return elapsed >= timeoutSeconds;
    }
};

// 타임아웃 체크 스레드 (간단한 예시)
void TimeoutChecker() {
    while (running) {
        Sleep(10000);  // 10초마다 체크
        
        auto sessions = g_sessionPool->GetActiveSessions();
        for (auto session : sessions) {
            if (session->IsTimeout(60)) {  // 60초 타임아웃
                std::cout << "세션 타임아웃: " << session->GetSessionId() 
                         << std::endl;
                session->Disconnect();
            }
        }
    }
}
```

## 3.6 전체 서버 코드
이제 Chapter 3에서 학습한 내용을 모두 종합하여 완전한 서버 코드를 구현한다. 이 코드는 `AcceptEx`를 이용한 비동기 Accept, 세션 풀 관리, 안전한 연결 해제를 모두 포함한다.

**Session.h**

```cpp
#pragma once

#include <WinSock2.h>
#include <Windows.h>
#include <atomic>
#include <chrono>

enum class SessionState {
    Idle,
    Connected,
    Disconnecting,
    Disconnected
};

class Session {
public:
    struct RecvOverlapped {
        OVERLAPPED overlapped;
        Session* owner;
        WSABUF wsaBuf;
    };

    struct SendOverlapped {
        OVERLAPPED overlapped;
        Session* owner;
        std::vector<BYTE> buffer;
    };

private:
    SOCKET socket_;
    uint64_t sessionId_;
    std::atomic<SessionState> state_;
    std::atomic<int32_t> refCount_;
    
    RecvOverlapped recvOverlapped_;
    BYTE recvBuffer_[4096];
    
    std::atomic<int32_t> sendInProgress_;
    bool pendingDisconnect_;
    
    std::chrono::steady_clock::time_point lastRecvTime_;

public:
    Session();
    ~Session();
    
    void Reset(SOCKET socket, uint64_t sessionId);
    void Initialize();
    void Disconnect();
    
    void PostRecv();
    void PostSend(const BYTE* data, int32_t len);
    
    void ProcessRecv(DWORD bytesTransferred);
    void ProcessSend(SendOverlapped* sendOv, DWORD bytesTransferred);
    
    void AddRef();
    void Release();
    
    SOCKET GetSocket() const { return socket_; }
    uint64_t GetSessionId() const { return sessionId_; }
    SessionState GetState() const { return state_.load(); }
    
    void UpdateRecvTime();
    bool IsTimeout(int32_t timeoutSeconds) const;
};
```

**Session.cpp**

```cpp
#include "Session.h"
#include "ServerGlobal.h"
#include <iostream>

Session::Session()
    : socket_(INVALID_SOCKET)
    , sessionId_(0)
    , state_(SessionState::Idle)
    , refCount_(0)
    , sendInProgress_(0)
    , pendingDisconnect_(false)
{
    ZeroMemory(&recvOverlapped_, sizeof(recvOverlapped_));
    recvOverlapped_.owner = this;
}

Session::~Session() {
    if (socket_ != INVALID_SOCKET) {
        closesocket(socket_);
    }
}

void Session::Reset(SOCKET socket, uint64_t sessionId) {
    socket_ = socket;
    sessionId_ = sessionId;
    state_ = SessionState::Idle;
    refCount_ = 1;
    sendInProgress_ = 0;
    pendingDisconnect_ = false;
    
    ZeroMemory(&recvOverlapped_.overlapped, sizeof(OVERLAPPED));
    
    UpdateRecvTime();
}

void Session::Initialize() {
    // IOCP에 소켓 연결
    CreateIoCompletionPort((HANDLE)socket_, g_iocpHandle,
                           (ULONG_PTR)this, 0);
    
    state_ = SessionState::Connected;
    
    // 첫 번째 비동기 수신 시작
    PostRecv();
    
    std::cout << "[Session] 세션 " << sessionId_ << " 연결됨" << std::endl;
}

void Session::Disconnect() {
    SessionState expected = SessionState::Connected;
    
    if (!state_.compare_exchange_strong(expected, SessionState::Disconnecting)) {
        return;  // 이미 연결 해제 중
    }
    
    std::cout << "[Session] 세션 " << sessionId_ << " 연결 해제" << std::endl;
    
    // 소켓 종료
    if (socket_ != INVALID_SOCKET) {
        closesocket(socket_);
        socket_ = INVALID_SOCKET;
    }
    
    state_ = SessionState::Disconnected;
    
    // 세션 풀에 반환
    g_sessionPool->Release(this);
    
    // 초기 참조 카운트 감소
    Release();
}

void Session::PostRecv() {
    if (state_ != SessionState::Connected)
        return;
    
    ZeroMemory(&recvOverlapped_.overlapped, sizeof(OVERLAPPED));
    
    recvOverlapped_.wsaBuf.buf = (char*)recvBuffer_;
    recvOverlapped_.wsaBuf.len = sizeof(recvBuffer_);
    
    DWORD flags = 0;
    DWORD bytesReceived = 0;
    
    AddRef();
    
    int result = WSARecv(socket_, &recvOverlapped_.wsaBuf, 1,
                         &bytesReceived, &flags,
                         &recvOverlapped_.overlapped, nullptr);
    
    if (result == SOCKET_ERROR) {
        int error = WSAGetLastError();
        if (error != WSA_IO_PENDING) {
            std::cout << "[Session] WSARecv 실패: " << error << std::endl;
            Release();
            Disconnect();
        }
    }
}

void Session::PostSend(const BYTE* data, int32_t len) {
    if (state_ != SessionState::Connected)
        return;
    
    SendOverlapped* sendOv = new SendOverlapped;
    ZeroMemory(&sendOv->overlapped, sizeof(OVERLAPPED));
    sendOv->owner = this;
    sendOv->buffer.assign(data, data + len);
    
    WSABUF wsaBuf;
    wsaBuf.buf = (char*)sendOv->buffer.data();
    wsaBuf.len = static_cast<ULONG>(sendOv->buffer.size());
    
    sendInProgress_.fetch_add(1);
    AddRef();
    
    DWORD bytesSent = 0;
    int result = WSASend(socket_, &wsaBuf, 1, &bytesSent, 0,
                         &sendOv->overlapped, nullptr);
    
    if (result == SOCKET_ERROR) {
        int error = WSAGetLastError();
        if (error != WSA_IO_PENDING) {
            std::cout << "[Session] WSASend 실패: " << error << std::endl;
            sendInProgress_.fetch_sub(1);
            Release();
            delete sendOv;
            Disconnect();
        }
    }
}

void Session::ProcessRecv(DWORD bytesTransferred) {
    UpdateRecvTime();
    
    if (bytesTransferred == 0) {
        // 정상 종료
        Disconnect();
        return;
    }
    
    // Echo 처리 (받은 데이터를 그대로 전송)
    PostSend(recvBuffer_, bytesTransferred);
    
    // 다음 수신 대기
    PostRecv();
}

void Session::ProcessSend(SendOverlapped* sendOv, DWORD bytesTransferred) {
    delete sendOv;
    
    int32_t sendCount = sendInProgress_.fetch_sub(1) - 1;
    
    // 송신 완료 후 연결 해제 대기 중이면
    if (pendingDisconnect_ && sendCount == 0) {
        Disconnect();
    }
}

void Session::AddRef() {
    refCount_.fetch_add(1);
}

void Session::Release() {
    int32_t refCount = refCount_.fetch_sub(1) - 1;
    if (refCount == 0) {
        // 참조 카운트가 0이 되면 완전히 해제
        // 세션 풀이 관리하므로 delete하지 않음
    }
}

void Session::UpdateRecvTime() {
    lastRecvTime_ = std::chrono::steady_clock::now();
}

bool Session::IsTimeout(int32_t timeoutSeconds) const {
    auto now = std::chrono::steady_clock::now();
    auto elapsed = std::chrono::duration_cast<std::chrono::seconds>(
        now - lastRecvTime_).count();
    return elapsed >= timeoutSeconds;
}
```

**SessionPool.cpp**

```cpp
#include "SessionPool.h"
#include <iostream>

SessionPool::SessionPool(int32_t maxSessions)
    : sessionIdGenerator_(1)
    , maxSessions_(maxSessions)
{
    for (int32_t i = 0; i < maxSessions_; ++i) {
        Session* session = new Session();
        freeList_.push(session);
    }
    
    std::cout << "[SessionPool] 세션 풀 생성: " << maxSessions_ 
              << "개" << std::endl;
}

SessionPool::~SessionPool() {
    // 모든 세션 해제
    while (!freeList_.empty()) {
        Session* session = freeList_.front();
        freeList_.pop();
        delete session;
    }
    
    for (auto& pair : activeSessions_) {
        delete pair.second;
    }
}

Session* SessionPool::Acquire(SOCKET socket) {
    Session* session = nullptr;
    
    {
        std::lock_guard<std::mutex> lock(freeListMutex_);
        
        if (freeList_.empty()) {
            std::cout << "[SessionPool] 세션 풀 고갈" << std::endl;
            return nullptr;
        }
        
        session = freeList_.front();
        freeList_.pop();
    }
    
    uint64_t sessionId = sessionIdGenerator_.fetch_add(1);
    session->Reset(socket, sessionId);
    
    {
        std::lock_guard<std::mutex> lock(activeSessionsMutex_);
        activeSessions_[sessionId] = session;
    }
    
    return session;
}

void SessionPool::Release(Session* session) {
    uint64_t sessionId = session->GetSessionId();
    
    {
        std::lock_guard<std::mutex> lock(activeSessionsMutex_);
        activeSessions_.erase(sessionId);
    }
    
    {
        std::lock_guard<std::mutex> lock(freeListMutex_);
        freeList_.push(session);
    }
}

std::vector<Session*> SessionPool::GetActiveSessions() {
    std::lock_guard<std::mutex> lock(activeSessionsMutex_);
    
    std::vector<Session*> sessions;
    sessions.reserve(activeSessions_.size());
    
    for (auto& pair : activeSessions_) {
        sessions.push_back(pair.second);
    }
    
    return sessions;
}

int32_t SessionPool::GetActiveSessionCount() {
    std::lock_guard<std::mutex> lock(activeSessionsMutex_);
    return static_cast<int32_t>(activeSessions_.size());
}
```

```cpp
#include "AcceptThread.h"
#include "ServerGlobal.h"
#include "Session.h"
#include <iostream>

LPFN_ACCEPTEX g_AcceptEx = nullptr;
LPFN_GETACCEPTEXSOCKADDRS g_GetAcceptExSockaddrs = nullptr;

bool InitializeAccept(const char* ip, uint16_t port) {
    // Listen 소켓 생성
    g_listenSocket = WSASocket(AF_INET, SOCK_STREAM, IPPROTO_TCP,
                               nullptr, 0, WSA_FLAG_OVERLAPPED);
    
    if (g_listenSocket == INVALID_SOCKET) {
        std::cout << "[Accept] Listen 소켓 생성 실패" << std::endl;
        return false;
    }
    
    // SO_REUSEADDR 설정
    BOOL reuseAddr = TRUE;
    setsockopt(g_listenSocket, SOL_SOCKET, SO_REUSEADDR,
               (char*)&reuseAddr, sizeof(reuseAddr));
    
    // Bind
    SOCKADDR_IN serverAddr;
    serverAddr.sin_family = AF_INET;
    serverAddr.sin_port = htons(port);
    inet_pton(AF_INET, ip, &serverAddr.sin_addr);
    
    if (bind(g_listenSocket, (SOCKADDR*)&serverAddr, 
             sizeof(serverAddr)) == SOCKET_ERROR) {
        std::cout << "[Accept] Bind 실패" << std::endl;
        return false;
    }
    
    // Listen
    if (listen(g_listenSocket, SOMAXCONN) == SOCKET_ERROR) {
        std::cout << "[Accept] Listen 실패" << std::endl;
        return false;
    }
    
    // Listen 소켓을 IOCP에 연결
    CreateIoCompletionPort((HANDLE)g_listenSocket, g_iocpHandle,
                           COMPLETION_KEY_ACCEPT, 0);
    
    // AcceptEx 함수 포인터 얻기
    GUID guidAcceptEx = WSAID_ACCEPTEX;
    DWORD bytes = 0;
    
    WSAIoctl(g_listenSocket,
             SIO_GET_EXTENSION_FUNCTION_POINTER,
             &guidAcceptEx, sizeof(guidAcceptEx),
             &g_AcceptEx, sizeof(g_AcceptEx),
             &bytes, nullptr, nullptr);
    
    if (g_AcceptEx == nullptr) {
        std::cout << "[Accept] AcceptEx 함수 포인터 획득 실패" << std::endl;
        return false;
    }
    
    // GetAcceptExSockaddrs 함수 포인터 얻기
    GUID guidGetAcceptExSockaddrs = WSAID_GETACCEPTEXSOCKADDRS;
    
    WSAIoctl(g_listenSocket,
             SIO_GET_EXTENSION_FUNCTION_POINTER,
             &guidGetAcceptExSockaddrs, sizeof(guidGetAcceptExSockaddrs),
             &g_GetAcceptExSockaddrs, sizeof(g_GetAcceptExSockaddrs),
             &bytes, nullptr, nullptr);
    
    if (g_GetAcceptExSockaddrs == nullptr) {
        std::cout << "[Accept] GetAcceptExSockaddrs 함수 포인터 획득 실패" 
                  << std::endl;
        return false;
    }
    
    // 여러 개의 Accept 작업 대기
    SYSTEM_INFO si;
    GetSystemInfo(&si);
    int prePostedAcceptCount = si.dwNumberOfProcessors * 2;
    
    for (int i = 0; i < prePostedAcceptCount; ++i) {
        PostAccept();
    }
    
    std::cout << "[Accept] 서버 시작: " << ip << ":" << port << std::endl;
    
    return true;
}

void PostAccept() {
    AcceptOverlapped* acceptOv = new AcceptOverlapped{};
    ZeroMemory(&acceptOv->overlapped, sizeof(OVERLAPPED));
    
    acceptOv->acceptSocket = WSASocket(AF_INET, SOCK_STREAM, IPPROTO_TCP,
                                       nullptr, 0, WSA_FLAG_OVERLAPPED);
    
    if (acceptOv->acceptSocket == INVALID_SOCKET) {
        std::cout << "[Accept] Accept 소켓 생성 실패" << std::endl;
        delete acceptOv;
        return;
    }
    
    DWORD bytes = 0;
    BOOL result = g_AcceptEx(
        g_listenSocket,
        acceptOv->acceptSocket,
        acceptOv->buffer,
        0,  // 첫 데이터를 받지 않음
        sizeof(SOCKADDR_STORAGE) + 16,
        sizeof(SOCKADDR_STORAGE) + 16,
        &bytes,
        &acceptOv->overlapped
    );
    
    if (!result && WSAGetLastError() != ERROR_IO_PENDING) {
        std::cout << "[Accept] AcceptEx 실패: " << WSAGetLastError() 
                  << std::endl;
        closesocket(acceptOv->acceptSocket);
        delete acceptOv;
    }
}

void ProcessAccept(AcceptOverlapped* acceptOv, DWORD bytesTransferred) {
    // Accept 컨텍스트 업데이트
    setsockopt(acceptOv->acceptSocket, SOL_SOCKET,
               SO_UPDATE_ACCEPT_CONTEXT,
               (char*)&g_listenSocket, sizeof(g_listenSocket));
    
    // 클라이언트 주소 정보 추출
    SOCKADDR* localAddr = nullptr;
    SOCKADDR* remoteAddr = nullptr;
    int localAddrLen = 0;
    int remoteAddrLen = 0;
    
    g_GetAcceptExSockaddrs(
        acceptOv->buffer,
        0,
        sizeof(SOCKADDR_STORAGE) + 16,
        sizeof(SOCKADDR_STORAGE) + 16,
        &localAddr, &localAddrLen,
        &remoteAddr, &remoteAddrLen
    );
    
    char clientIP[INET_ADDRSTRLEN];
    inet_ntop(AF_INET, &((SOCKADDR_IN*)remoteAddr)->sin_addr,
              clientIP, sizeof(clientIP));
    
    std::cout << "[Accept] 새 연결: " << clientIP << std::endl;
    
    // 세션 풀에서 세션 획득
    Session* session = g_sessionPool->Acquire(acceptOv->acceptSocket);
    
    if (session != nullptr) {
        session->Initialize();
    } else {
        std::cout << "[Accept] 세션 풀 고갈, 연결 거부" << std::endl;
        closesocket(acceptOv->acceptSocket);
    }
    
    // 새로운 Accept 대기
    PostAccept();
    
    delete acceptOv;
}
```

**WorkerThread.cpp**

```cpp
#include "WorkerThread.h"
#include "ServerGlobal.h"
#include "Session.h"
#include "AcceptThread.h"
#include <iostream>

DWORD WINAPI WorkerThread(LPVOID param) {
    while (g_running) {
        DWORD bytesTransferred = 0;
        ULONG_PTR completionKey = 0;
        LPOVERLAPPED overlapped = nullptr;
        
        BOOL result = GetQueuedCompletionStatus(
            g_iocpHandle,
            &bytesTransferred,
            &completionKey,
            &overlapped,
            INFINITE
        );
        
        // 종료 신호
        if (completionKey == 0 && overlapped == nullptr) {
            break;
        }
        
        // Accept 완료
        if (completionKey == COMPLETION_KEY_ACCEPT) {
            AcceptOverlapped* acceptOv = (AcceptOverlapped*)overlapped;
            ProcessAccept(acceptOv, bytesTransferred);
            continue;
        }
        
        // 세션 I/O 완료
        Session* session = (Session*)completionKey;
        
        if (result == FALSE || bytesTransferred == 0) {
            // 연결 해제
            session->Disconnect();
            session->Release();
            continue;
        }
        
        // Recv 완료
        if (overlapped == &session->recvOverlapped_.overlapped) {
            session->ProcessRecv(bytesTransferred);
            session->Release();
        }
        // Send 완료
        else {
            Session::SendOverlapped* sendOv = 
                (Session::SendOverlapped*)overlapped;
            session->ProcessSend(sendOv, bytesTransferred);
            session->Release();
        }
    }
    
    return 0;
}
```

```cpp
#include <iostream>
#include <vector>
#include "ServerGlobal.h"
#include "WorkerThread.h"
#include "AcceptThread.h"

int main() {
    std::cout << "=== IOCP 게임 서버 Chapter 3 ===" << std::endl;
    
    // 서버 초기화
    if (!InitializeServer()) {
        std::cout << "서버 초기화 실패" << std::endl;
        return -1;
    }
    
    // Accept 초기화
    if (!InitializeAccept("0.0.0.0", 9000)) {
        std::cout << "Accept 초기화 실패" << std::endl;
        ShutdownServer();
        return -1;
    }
    
    // 워커 스레드 생성
    SYSTEM_INFO si;
    GetSystemInfo(&si);
    int workerThreadCount = si.dwNumberOfProcessors * 2;
    
    std::vector<HANDLE> workerThreads;
    for (int i = 0; i < workerThreadCount; ++i) {
        HANDLE thread = CreateThread(nullptr, 0, WorkerThread, nullptr, 0, nullptr);
        workerThreads.push_back(thread);
    }
    
    std::cout << "워커 스레드 " << workerThreadCount << "개 생성됨" << std::endl;
    std::cout << "서버 실행 중... (종료하려면 'q' 입력)" << std::endl;
    
    // 메인 루프
    while (g_running) {
        char input;
        std::cin >> input;
        
        if (input == 'q' || input == 'Q') {
            g_running = false;
            break;
        }
        
        if (input == 's' || input == 'S') {
            int activeCount = g_sessionPool->GetActiveSessionCount();
            std::cout << "현재 활성 세션: " << activeCount << "개" << std::endl;
        }
    }
    
    std::cout << "서버 종료 중..." << std::endl;
    
    // 워커 스레드 종료 신호
    for (size_t i = 0; i < workerThreads.size(); ++i) {
        PostQueuedCompletionStatus(g_iocpHandle, 0, 0, nullptr);
    }
    
    // 워커 스레드 종료 대기
    WaitForMultipleObjects(static_cast<DWORD>(workerThreads.size()),
                          workerThreads.data(), TRUE, INFINITE);
    
    for (HANDLE thread : workerThreads) {
        CloseHandle(thread);
    }
    
    // 서버 종료
    ShutdownServer();
    
    std::cout << "서버 종료 완료" << std::endl;
    
    return 0;
}
```
