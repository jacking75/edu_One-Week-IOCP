# Chapter 5. 패킷 설계와 직렬화
Chapter 4에서 링 버퍼를 구현하여 효율적인 송수신 버퍼 관리를 완성했다. 하지만 단순히 바이트 스트림을 주고받는 것만으로는 의미 있는 통신을 할 수 없다. 게임 서버와 클라이언트는 구조화된 데이터를 주고받아야 하며, 이를 위해서는 체계적인 패킷 설계가 필요하다.

패킷 설계는 게임 서버 개발에서 가장 기초가 되는 부분이면서도 가장 중요한 부분이다. 잘못 설계된 패킷 구조는 나중에 수정하기 매우 어렵고, 성능 문제나 보안 취약점의 원인이 될 수 있다. 이번 챕터에서는 패킷의 구조를 설계하고, 데이터를 바이트 스트림으로 변환하는 직렬화(Serialization)와 그 반대 과정인 역직렬화(Deserialization)를 구현한다. 또한 네트워크 통신에서 발생하는 엔디안 문제와 바이트 정렬 문제도 다룬다.

## 5.1 패킷 구조 설계 (헤더 + 바디)

**패킷의 기본 구조**  
네트워크 패킷은 일반적으로 헤더(Header)와 바디(Body)로 구성된다. 헤더는 패킷의 메타 정보를 담고 있으며, 바디는 실제 전송하려는 데이터를 포함한다. 헤더는 고정 크기로 설계하는 것이 일반적이며, 바디는 패킷 종류에 따라 가변 크기를 가질 수 있다.

```
[패킷 구조]

┌─────────────────────────────────────┐
│           Packet Header             │
│  ┌──────────┬──────────┬─────────┐ │
│  │  Size    │ PacketID │ Reserved│ │
│  │ (2 byte) │ (2 byte) │ (4 byte)│ │
│  └──────────┴──────────┴─────────┘ │
├─────────────────────────────────────┤
│           Packet Body               │
│  ┌─────────────────────────────┐   │
│  │   Payload Data              │   │
│  │   (Variable Length)         │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘
```

헤더에 포함되어야 할 기본적인 정보는 다음과 같다. 첫째, 패킷의 전체 크기다. 이를 통해 수신 측에서 패킷의 경계를 구분할 수 있다. 둘째, 패킷의 타입이나 ID다. 이를 통해 어떤 종류의 패킷인지 식별하고 적절한 핸들러로 라우팅할 수 있다. 셋째, 선택적으로 시퀀스 번호, 체크섬, 암호화 플래그 등의 추가 정보를 포함할 수 있다.

**고정 헤더 설계**  
가장 단순하면서도 효율적인 방법은 고정 크기의 헤더를 사용하는 것이다. 고정 헤더는 파싱이 간단하고 성능이 좋으며, 구현이 단순하다는 장점이 있다. 우리는 8바이트 고정 헤더를 설계한다.

```cpp
#pragma pack(push, 1)
struct PacketHeader {
    uint16_t size;       // 헤더 포함 전체 패킷 크기 (최대 65535)
    uint16_t packetId;   // 패킷 타입 ID
    uint32_t reserved;   // 예약 필드 (체크섬, 시퀀스 등에 활용)
};
#pragma pack(pop)

static_assert(sizeof(PacketHeader) == 8, "PacketHeader must be 8 bytes");
```

`#pragma pack(push, 1)`은 구조체의 바이트 정렬(alignment)을 1바이트로 강제한다. 이를 통해 컴파일러가 구조체 멤버 사이에 패딩(padding)을 추가하지 않도록 한다. 네트워크 통신에서는 바이트 하나하나가 정확히 일치해야 하므로 패딩이 있으면 안 된다.

`static_assert`는 컴파일 타임에 조건을 검사한다. 만약 구조체 크기가 8바이트가 아니면 컴파일 에러가 발생하여, 의도치 않은 패딩이나 정렬 문제를 미리 발견할 수 있다.

**reserved 필드의 활용**  
reserved 필드는 미래의 확장성을 위해 예약해둔 공간이다. 초기에는 0으로 설정하고 사용하지 않지만, 나중에 다음과 같은 용도로 활용할 수 있다.

```cpp
// reserved 필드를 비트 플래그로 활용하는 예
union ReservedField {
    uint32_t value;
    struct {
        uint32_t compressed : 1;   // 압축 여부
        uint32_t encrypted : 1;    // 암호화 여부
        uint32_t sequence : 16;    // 시퀀스 번호
        uint32_t checksum : 14;    // 체크섬
    } flags;
};
```

**패킷 크기 제한**  
패킷 크기는 `uint16_t`로 표현하므로 이론적으로 최대 65535바이트까지 가능하다. 하지만 실제로는 더 작은 크기로 제한하는 것이 좋다. 너무 큰 패킷은 처리 시간이 길어지고, 메모리 사용량이 증가하며, DoS 공격에 취약해진다.

```cpp
// 패킷 크기 제한
constexpr uint16_t MIN_PACKET_SIZE = sizeof(PacketHeader);
constexpr uint16_t MAX_PACKET_SIZE = 8192;  // 8KB

// 패킷 크기 검증
bool IsValidPacketSize(uint16_t size) {
    return size >= MIN_PACKET_SIZE && size <= MAX_PACKET_SIZE;
}
```

일반적인 게임에서는 4KB~8KB 정도가 적절하다. 대부분의 게임 패킷은 수백 바이트 이내이므로, 8KB면 충분히 여유가 있다. 만약 더 큰 데이터를 전송해야 한다면 패킷을 여러 개로 분할하거나, 별도의 파일 전송 프로토콜을 사용하는 것이 좋다.

## 5.2 패킷 ID와 크기 정의

**패킷 ID 체계**  
패킷 ID는 패킷의 종류를 구분하는 식별자다. 체계적으로 관리하지 않으면 중복되거나 누락될 수 있으므로, enum class를 사용하여 타입 안전하게 관리한다.

```cpp
enum class PacketID : uint16_t {
    // 시스템 패킷 (0~999)
    NONE = 0,
    HEARTBEAT = 1,
    HEARTBEAT_ACK = 2,
    
    // 인증 패킷 (1000~1999)
    LOGIN_REQ = 1000,
    LOGIN_RES = 1001,
    LOGOUT_REQ = 1002,
    LOGOUT_RES = 1003,
    
    // 게임 패킷 (2000~2999)
    ENTER_ROOM_REQ = 2000,
    ENTER_ROOM_RES = 2001,
    LEAVE_ROOM_REQ = 2002,
    LEAVE_ROOM_RES = 2003,
    
    CHAT_REQ = 2100,
    CHAT_BROADCAST = 2101,
    
    // 오목 게임 패킷 (3000~3999)
    OMOK_READY_REQ = 3000,
    OMOK_READY_RES = 3001,
    OMOK_MOVE_REQ = 3002,
    OMOK_MOVE_BROADCAST = 3003,
    OMOK_RESULT = 3004,
    
    // 최대값
    MAX_PACKET_ID = 65535
};
```

패킷 ID를 범위별로 그룹화하면 관리가 쉽고, 나중에 새로운 패킷을 추가할 때도 적절한 위치에 넣을 수 있다. 또한 범위를 보고 패킷의 카테고리를 쉽게 파악할 수 있다.

**패킷 구조 정의**  
각 패킷 타입마다 구조체를 정의한다. 모든 패킷은 `PacketHeader`를 첫 번째 멤버로 가지며, 그 뒤에 패킷별 데이터가 온다.

```cpp
#pragma pack(push, 1)

// 로그인 요청 패킷
struct PKT_LOGIN_REQ {
    PacketHeader header;
    char accountId[20];
    char password[20];
    
    PKT_LOGIN_REQ() {
        header.size = sizeof(PKT_LOGIN_REQ);
        header.packetId = static_cast<uint16_t>(PacketID::LOGIN_REQ);
        header.reserved = 0;
    }
};

// 로그인 응답 패킷
struct PKT_LOGIN_RES {
    PacketHeader header;
    uint8_t result;      // 0: 성공, 1: 실패
    uint64_t playerId;
    char playerName[20];
    
    PKT_LOGIN_RES() {
        header.size = sizeof(PKT_LOGIN_RES);
        header.packetId = static_cast<uint16_t>(PacketID::LOGIN_RES);
        header.reserved = 0;
    }
};

// 채팅 패킷
struct PKT_CHAT_REQ {
    PacketHeader header;
    uint16_t messageLen;
    char message[256];
    
    PKT_CHAT_REQ() {
        header.size = sizeof(PacketHeader) + sizeof(messageLen);
        header.packetId = static_cast<uint16_t>(PacketID::CHAT_REQ);
        header.reserved = 0;
        messageLen = 0;
    }
    
    void SetMessage(const char* msg) {
        size_t len = strlen(msg);
        if (len > 255) len = 255;
        messageLen = static_cast<uint16_t>(len);
        memcpy(message, msg, len);
        message[len] = '\0';
        header.size = sizeof(PacketHeader) + sizeof(messageLen) + messageLen;
    }
};

#pragma pack(pop)
```

**가변 길이 데이터 처리**  
채팅 메시지처럼 길이가 가변적인 데이터는 두 가지 방법으로 처리할 수 있다. 첫 번째는 최대 크기의 배열을 선언하고 실제 사용하는 길이를 별도로 저장하는 방법이다. 두 번째는 데이터를 패킷 뒤에 붙이고 길이 정보를 저장하는 방법이다.

```cpp
// 방법 1: 고정 크기 배열 (위의 PKT_CHAT_REQ)
// 장점: 구조체로 직접 접근 가능, 간단함
// 단점: 메모리 낭비 가능

// 방법 2: 뒤에 붙이는 방식
struct PKT_CHAT_REQ_V2 {
    PacketHeader header;
    uint16_t messageLen;
    // message는 구조체 뒤에 이어짐
    
    char* GetMessage() {
        return reinterpret_cast<char*>(this + 1);
    }
    
    static PKT_CHAT_REQ_V2* Create(const char* msg) {
        size_t len = strlen(msg);
        if (len > 1024) len = 1024;
        
        size_t totalSize = sizeof(PKT_CHAT_REQ_V2) + len;
        PKT_CHAT_REQ_V2* pkt = 
            reinterpret_cast<PKT_CHAT_REQ_V2*>(new BYTE[totalSize]);
        
        pkt->header.size = static_cast<uint16_t>(totalSize);
        pkt->header.packetId = static_cast<uint16_t>(PacketID::CHAT_REQ);
        pkt->header.reserved = 0;
        pkt->messageLen = static_cast<uint16_t>(len);
        memcpy(pkt->GetMessage(), msg, len);
        
        return pkt;
    }
};
```

방법 1은 구현이 간단하고 안전하지만 메모리 낭비가 있을 수 있다. 방법 2는 메모리 효율적이지만 동적 할당이 필요하고 구현이 복잡하다. 일반적으로 방법 1을 사용하되, 매우 큰 데이터는 방법 2를 사용하는 것이 좋다.

## 5.3 직렬화(Serialization)와 역직렬화(Deserialization)  
  
**직렬화의 필요성**  
구조체를 그대로 네트워크로 전송하는 것은 간단하지만 여러 문제가 있다. 컴파일러마다 바이트 정렬이 다를 수 있고, 엔디안이 다른 시스템 간에는 데이터가 깨진다. 또한 구조체에 포인터가 있으면 전송할 수 없다. 직렬화는 이러한 문제를 해결하기 위해 데이터를 표준화된 바이트 스트림으로 변환하는 과정이다.

간단한 게임 서버에서는 구조체를 직접 전송하는 방식도 충분히 사용할 수 있다. 하지만 더 안전하고 유연한 방법은 PacketWriter/PacketReader 클래스를 만들어 명시적으로 직렬화/역직렬화하는 것이다.

**PacketWriter 구현**  
PacketWriter는 데이터를 바이트 스트림으로 변환하여 버퍼에 쓰는 클래스다. 다양한 타입의 데이터를 순차적으로 쓸 수 있도록 설계한다.

```cpp
class PacketWriter {
private:
    std::vector<BYTE> buffer_;
    size_t writePos_;
    
public:
    PacketWriter(size_t reserveSize = 256)
        : writePos_(0)
    {
        buffer_.reserve(reserveSize);
        // 헤더 공간 예약
        buffer_.resize(sizeof(PacketHeader));
        writePos_ = sizeof(PacketHeader);
    }
    
    // 기본 타입 쓰기
    template<typename T>
    void Write(const T& value) {
        static_assert(std::is_trivially_copyable_v<T>, 
                     "T must be trivially copyable");
        
        size_t oldSize = buffer_.size();
        buffer_.resize(oldSize + sizeof(T));
        memcpy(&buffer_[writePos_], &value, sizeof(T));
        writePos_ += sizeof(T);
    }
    
    // 문자열 쓰기 (길이 + 데이터)
    void WriteString(const std::string& str) {
        uint16_t len = static_cast<uint16_t>(str.length());
        Write(len);
        
        if (len > 0) {
            size_t oldSize = buffer_.size();
            buffer_.resize(oldSize + len);
            memcpy(&buffer_[writePos_], str.c_str(), len);
            writePos_ += len;
        }
    }
    
    // 고정 길이 문자열 쓰기
    void WriteFixedString(const char* str, size_t maxLen) {
        size_t len = strnlen(str, maxLen);
        size_t oldSize = buffer_.size();
        buffer_.resize(oldSize + maxLen);
        
        memcpy(&buffer_[writePos_], str, len);
        if (len < maxLen) {
            memset(&buffer_[writePos_ + len], 0, maxLen - len);
        }
        writePos_ += maxLen;
    }
    
    // 바이트 배열 쓰기
    void WriteBytes(const BYTE* data, size_t size) {
        size_t oldSize = buffer_.size();
        buffer_.resize(oldSize + size);
        memcpy(&buffer_[writePos_], data, size);
        writePos_ += size;
    }
    
    // 패킷 완성
    BYTE* Finalize(PacketID packetId) {
        PacketHeader* header = reinterpret_cast<PacketHeader*>(&buffer_[0]);
        header->size = static_cast<uint16_t>(buffer_.size());
        header->packetId = static_cast<uint16_t>(packetId);
        header->reserved = 0;
        
        return buffer_.data();
    }
    
    size_t GetSize() const { return buffer_.size(); }
    const BYTE* GetData() const { return buffer_.data(); }
};
```

**PacketReader 구현**  
PacketReader는 바이트 스트림에서 데이터를 읽어 역직렬화하는 클래스다. 읽기 위치를 추적하고, 버퍼 오버런을 방지한다.

```cpp
class PacketReader {
private:
    const BYTE* buffer_;
    size_t bufferSize_;
    size_t readPos_;
    
public:
    PacketReader(const BYTE* buffer, size_t size)
        : buffer_(buffer)
        , bufferSize_(size)
        , readPos_(sizeof(PacketHeader))  // 헤더는 건너뜀
    {}
    
    // 헤더 읽기
    const PacketHeader* GetHeader() const {
        return reinterpret_cast<const PacketHeader*>(buffer_);
    }
    
    // 기본 타입 읽기
    template<typename T>
    bool Read(T& outValue) {
        static_assert(std::is_trivially_copyable_v<T>,
                     "T must be trivially copyable");
        
        if (readPos_ + sizeof(T) > bufferSize_) {
            return false;  // 버퍼 오버런
        }
        
        memcpy(&outValue, &buffer_[readPos_], sizeof(T));
        readPos_ += sizeof(T);
        return true;
    }
    
    // 문자열 읽기
    bool ReadString(std::string& outStr) {
        uint16_t len;
        if (!Read(len)) {
            return false;
        }
        
        if (readPos_ + len > bufferSize_) {
            return false;
        }
        
        outStr.assign(reinterpret_cast<const char*>(&buffer_[readPos_]), len);
        readPos_ += len;
        return true;
    }
    
    // 고정 길이 문자열 읽기
    bool ReadFixedString(char* outStr, size_t maxLen) {
        if (readPos_ + maxLen > bufferSize_) {
            return false;
        }
        
        memcpy(outStr, &buffer_[readPos_], maxLen);
        readPos_ += maxLen;
        return true;
    }
    
    // 바이트 배열 읽기
    bool ReadBytes(BYTE* outData, size_t size) {
        if (readPos_ + size > bufferSize_) {
            return false;
        }
        
        memcpy(outData, &buffer_[readPos_], size);
        readPos_ += size;
        return true;
    }
    
    // 남은 데이터 크기
    size_t GetRemainSize() const {
        return bufferSize_ - readPos_;
    }
    
    bool IsEnd() const {
        return readPos_ >= bufferSize_;
    }
};
```

**사용 예제**  
PacketWriter와 PacketReader를 사용하여 패킷을 직렬화/역직렬화하는 예제다.

```cpp
// 로그인 요청 패킷 작성
void SendLoginRequest(Session* session, 
                     const std::string& accountId,
                     const std::string& password) {
    PacketWriter writer;
    writer.WriteString(accountId);
    writer.WriteString(password);
    
    BYTE* data = writer.Finalize(PacketID::LOGIN_REQ);
    session->Send(data, writer.GetSize());
}

// 로그인 요청 패킷 파싱
void HandleLoginRequest(Session* session, const BYTE* data, size_t size) {
    PacketReader reader(data, size);
    
    std::string accountId, password;
    if (!reader.ReadString(accountId) || 
        !reader.ReadString(password)) {
        std::cout << "로그인 패킷 파싱 실패" << std::endl;
        return;
    }
    
    std::cout << "로그인 요청: " << accountId << std::endl;
    
    // 로그인 처리 로직...
    bool success = ProcessLogin(accountId, password);
    
    // 로그인 응답 전송
    SendLoginResponse(session, success);
}
```

## 5.4 엔디안(Endian) 문제 처리

**엔디안이란**  
엔디안은 멀티바이트 데이터를 메모리에 저장하는 순서를 의미한다. 빅 엔디안(Big Endian)은 상위 바이트를 낮은 주소에 저장하고, 리틀 엔디안(Little Endian)은 하위 바이트를 낮은 주소에 저장한다. 예를 들어 0x12345678이라는 4바이트 정수는 다음과 같이 저장된다.

```
Big Endian (네트워크 바이트 순서):
주소:  0x00  0x01  0x02  0x03
값:    0x12  0x34  0x56  0x78

Little Endian (Intel x86/x64):
주소:  0x00  0x01  0x02  0x03
값:    0x78  0x56  0x34  0x12
```

네트워크 통신에서는 빅 엔디안을 표준으로 사용한다. 이를 네트워크 바이트 순서(Network Byte Order)라고 부른다. Windows PC는 리틀 엔디안을 사용하므로, 네트워크로 데이터를 보낼 때는 엔디안 변환이 필요하다.

**엔디안 변환 함수**  
Windows에서는 `htons`, `htonl`, `ntohs`, `ntohl` 함수를 제공한다. h는 host, n은 network를 의미하고, s는 short(2바이트), l은 long(4바이트)를 의미한다.

```cpp
// 엔디안 변환 유틸리티
class EndianConverter {
public:
    // Host to Network
    static uint16_t HostToNetwork(uint16_t value) {
        return htons(value);
    }
    
    static uint32_t HostToNetwork(uint32_t value) {
        return htonl(value);
    }
    
    static uint64_t HostToNetwork(uint64_t value) {
        return _byteswap_uint64(value);
    }
    
    // Network to Host
    static uint16_t NetworkToHost(uint16_t value) {
        return ntohs(value);
    }
    
    static uint32_t NetworkToHost(uint32_t value) {
        return ntohl(value);
    }
    
    static uint64_t NetworkToHost(uint64_t value) {
        return _byteswap_uint64(value);
    }
    
    // 현재 시스템이 리틀 엔디안인지 확인
    static bool IsLittleEndian() {
        uint16_t test = 0x0001;
        return *reinterpret_cast<uint8_t*>(&test) == 0x01;
    }
};
```

**패킷에 엔디안 변환 적용**  
같은 플랫폼(Windows 끼리)에서만 통신한다면 엔디안 변환을 생략할 수 있다. 하지만 크로스 플랫폼을 지원하거나, 표준을 준수하려면 엔디안 변환을 적용해야 한다.

```cpp
#pragma pack(push, 1)
struct PacketHeader {
    uint16_t size;
    uint16_t packetId;
    uint32_t reserved;
    
    // 네트워크 바이트 순서로 변환
    void ToNetworkOrder() {
        size = EndianConverter::HostToNetwork(size);
        packetId = EndianConverter::HostToNetwork(packetId);
        reserved = EndianConverter::HostToNetwork(reserved);
    }
    
    // 호스트 바이트 순서로 변환
    void ToHostOrder() {
        size = EndianConverter::NetworkToHost(size);
        packetId = EndianConverter::NetworkToHost(packetId);
        reserved = EndianConverter::NetworkToHost(reserved);
    }
};
#pragma pack(pop)
```

실전에서는 성능을 위해 같은 플랫폼끼리 통신할 때는 엔디안 변환을 생략하는 경우가 많다. Windows 게임 서버와 Windows 클라이언트만 지원한다면 엔디안 변환 없이 구조체를 직접 전송해도 문제없다. 하지만 코드의 이식성을 고려한다면 처음부터 엔디안 변환을 적용하는 것이 좋다.

**PacketWriter/Reader에 엔디안 변환 적용**  
PacketWriter와 PacketReader에 엔디안 변환 기능을 추가할 수 있다. 템플릿 특수화를 사용하여 정수 타입에만 자동으로 변환을 적용한다.

```cpp
template<typename T>
concept IntegerType = std::is_integral_v<T> && !std::is_same_v<T, bool>;

class PacketWriter {
public:
    // 정수 타입은 엔디안 변환
    template<IntegerType T>
    void Write(T value) {
        if constexpr (sizeof(T) == 2) {
            value = static_cast<T>(EndianConverter::HostToNetwork(
                static_cast<uint16_t>(value)));
        } else if constexpr (sizeof(T) == 4) {
            value = static_cast<T>(EndianConverter::HostToNetwork(
                static_cast<uint32_t>(value)));
        } else if constexpr (sizeof(T) == 8) {
            value = static_cast<T>(EndianConverter::HostToNetwork(
                static_cast<uint64_t>(value)));
        }
        
        WriteRaw(&value, sizeof(T));
    }
    
    // POD 타입은 그대로 쓰기
    template<typename T>
    requires std::is_trivially_copyable_v<T> && (!IntegerType<T>)
    void Write(const T& value) {
        WriteRaw(&value, sizeof(T));
    }
    
private:
    void WriteRaw(const void* data, size_t size) {
        size_t oldSize = buffer_.size();
        buffer_.resize(oldSize + size);
        memcpy(&buffer_[writePos_], data, size);
        writePos_ += size;
    }
};
```

## 5.5 패킷 버퍼 클래스 구현

**SharedPacketBuffer 설계**  
패킷 데이터를 여러 세션에 브로드캐스트할 때, 각 세션마다 패킷을 복사하면 비효율적이다. `shared_ptr`을 사용하여 하나의 패킷 버퍼를 여러 세션이 공유하도록 설계한다.

```cpp
class PacketBuffer {
private:
    std::vector<BYTE> buffer_;
    
public:
    PacketBuffer(size_t size) : buffer_(size) {}
    
    BYTE* Data() { return buffer_.data(); }
    const BYTE* Data() const { return buffer_.data(); }
    size_t Size() const { return buffer_.size(); }
    
    // 패킷 헤더 접근
    PacketHeader* GetHeader() {
        return reinterpret_cast<PacketHeader*>(buffer_.data());
    }
    
    const PacketHeader* GetHeader() const {
        return reinterpret_cast<const PacketHeader*>(buffer_.data());
    }
};

using PacketBufferPtr = std::shared_ptr<PacketBuffer>;

// 패킷 버퍼 생성 헬퍼
class PacketBufferFactory {
public:
    static PacketBufferPtr Create(size_t size) {
        return std::make_shared<PacketBuffer>(size);
    }
    
    static PacketBufferPtr CreateFrom(const BYTE* data, size_t size) {
        auto buffer = Create(size);
        memcpy(buffer->Data(), data, size);
        return buffer;
    }
};
```

**PacketBuilder 패턴** 
빌더 패턴을 사용하면 패킷을 더 직관적으로 생성할 수 있다. 메서드 체이닝을 통해 가독성 좋은 코드를 작성할 수 있다.

```cpp
class PacketBuilder {
private:
    PacketWriter writer_;
    PacketID packetId_;
    
public:
    explicit PacketBuilder(PacketID id) : packetId_(id) {}
    
    PacketBuilder& AddUInt8(uint8_t value) {
        writer_.Write(value);
        return *this;
    }
    
    PacketBuilder& AddUInt16(uint16_t value) {
        writer_.Write(value);
        return *this;
    }
    
    PacketBuilder& AddUInt32(uint32_t value) {
        writer_.Write(value);
        return *this;
    }
    
    PacketBuilder& AddUInt64(uint64_t value) {
        writer_.Write(value);
        return *this;
    }
    
    PacketBuilder& AddString(const std::string& str) {
        writer_.WriteString(str);
        return *this;
    }
    
    PacketBuilder& AddFixedString(const char* str, size_t len) {
        writer_.WriteFixedString(str, len);
        return *this;
    }
    
    PacketBufferPtr Build() {
        BYTE* data = writer_.Finalize(packetId_);
        return PacketBufferFactory::CreateFrom(data, writer_.GetSize());
    }

        .AddUInt64(playerId)
        .AddString(message)
        .Build();
}
```

**패킷 풀(Packet Pool)**  
빈번하게 생성/삭제되는 패킷 버퍼는 풀링하여 재사용하면 성능이 향상된다. Concurrency Runtime의 `concurrent_queue`를 사용하여 스레드 안전한 패킷 풀을 구현한다.

```cpp
#include <concurrent_queue.h>

class PacketBufferPool {
private:
    concurrency::concurrent_queue<PacketBufferPtr> pool_;
    const size_t bufferSize_;
    std::atomic<int> allocCount_;
    std::atomic<int> poolCount_;
    
public:
    PacketBufferPool(size_t bufferSize, size_t preAllocCount = 100)
        : bufferSize_(bufferSize)
        , allocCount_(0)
        , poolCount_(0)
    {
        // 미리 할당
        for (size_t i = 0; i < preAllocCount; ++i) {
            pool_.push(std::make_shared<PacketBuffer>(bufferSize_));
            poolCount_++;
        }
    }
    
    PacketBufferPtr Acquire() {
        PacketBufferPtr buffer;
        
        if (pool_.try_pop(buffer)) {
            poolCount_--;
            return buffer;
        }
        
        // 풀이 비었으면 새로 할당
        allocCount_++;
        return std::make_shared<PacketBuffer>(bufferSize_);
    }
    
    void Release(PacketBufferPtr buffer) {
        if (buffer && buffer.use_count() == 1) {
            pool_.push(buffer);
            poolCount_++;
        }
    }
    
    void PrintStats() {
        std::cout << "패킷 버퍼 풀 통계:" << std::endl;
        std::cout << "  총 할당: " << allocCount_.load() << std::endl;
        std::cout << "  풀 크기: " << poolCount_.load() << std::endl;
    }
};

// 전역 패킷 풀
inline PacketBufferPool g_packetPool(MAX_PACKET_SIZE);
```

```cpp
#pragma once

#include <vector>
#include <string>
#include <cstring>
#include <type_traits>
#include "PacketTypes.h"

class PacketWriter {
private:
    std::vector<BYTE> buffer_;
    size_t writePos_;
    
public:
    PacketWriter(size_t reserveSize = 256)
        : writePos_(0)
    {
        buffer_.reserve(reserveSize);
        buffer_.resize(sizeof(PacketHeader));
        writePos_ = sizeof(PacketHeader);
    }
    
    // 기본 타입 쓰기
    template<typename T>
    void Write(const T& value) {
        static_assert(std::is_trivially_copyable_v<T>,
                     "T must be trivially copyable");
        
        size_t oldSize = buffer_.size();
        buffer_.resize(oldSize + sizeof(T));
        memcpy(&buffer_[writePos_], &value, sizeof(T));
        writePos_ += sizeof(T);
    }
    
    // 문자열 쓰기 (길이 + 데이터)
    void WriteString(const std::string& str) {
        uint16_t len = static_cast<uint16_t>(
            std::min(str.length(), size_t(1024)));
        Write(len);
        
        if (len > 0) {
            size_t oldSize = buffer_.size();
            buffer_.resize(oldSize + len);
            memcpy(&buffer_[writePos_], str.c_str(), len);
            writePos_ += len;
        }
    }
    
    // 고정 길이 문자열 쓰기
    void WriteFixedString(const char* str, size_t maxLen) {
        size_t len = strnlen(str, maxLen);
        size_t oldSize = buffer_.size();
        buffer_.resize(oldSize + maxLen);
        
        memcpy(&buffer_[writePos_], str, len);
        if (len < maxLen) {
            memset(&buffer_[writePos_ + len], 0, maxLen - len);
        }
        writePos_ += maxLen;
    }
    
    // 바이트 배열 쓰기
    void WriteBytes(const BYTE* data, size_t size) {
        size_t oldSize = buffer_.size();
        buffer_.resize(oldSize + size);
        memcpy(&buffer_[writePos_], data, size);
        writePos_ += size;
    }
    
    // 패킷 완성
    BYTE* Finalize(PacketID packetId) {
        PacketHeader* header = reinterpret_cast<PacketHeader*>(&buffer_[0]);
        header->size = static_cast<uint16_t>(buffer_.size());
        header->packetId = static_cast<uint16_t>(packetId);
        header->reserved = 0;
        
        return buffer_.data();
    }
    
    size_t GetSize() const { return buffer_.size(); }
    const BYTE* GetData() const { return buffer_.data(); }
    std::vector<BYTE>&& MoveBuffer() { return std::move(buffer_); }
};
```

**PacketReader.h**

```cpp
#pragma once

#include <string>
#include <cstring>
#include <type_traits>
#include "PacketTypes.h"

class PacketReader {
private:
    const BYTE* buffer_;
    size_t bufferSize_;
    size_t readPos_;
    bool error_;
    
public:
    PacketReader(const BYTE* buffer, size_t size)
        : buffer_(buffer)
        , bufferSize_(size)
        , readPos_(sizeof(PacketHeader))
        , error_(false)
    {
        if (size < sizeof(PacketHeader)) {
            error_ = true;
        }
    }
    
    // 헤더 읽기
    const PacketHeader* GetHeader() const {
        if (bufferSize_ < sizeof(PacketHeader)) {
            return nullptr;
        }
        return reinterpret_cast<const PacketHeader*>(buffer_);
    }
    
    // 기본 타입 읽기
    template<typename T>
    bool Read(T& outValue) {
        static_assert(std::is_trivially_copyable_v<T>,
                     "T must be trivially copyable");
        
        if (error_ || readPos_ + sizeof(T) > bufferSize_) {
            error_ = true;
            return false;
        }
        
        memcpy(&outValue, &buffer_[readPos_], sizeof(T));
        readPos_ += sizeof(T);
        return true;
    }
    
    // 문자열 읽기
    bool ReadString(std::string& outStr) {
        uint16_t len;
        if (!Read(len)) {
            return false;
        }
        
        if (error_ || readPos_ + len > bufferSize_) {
            error_ = true;
            return false;
        }
        
        outStr.assign(reinterpret_cast<const char*>(&buffer_[readPos_]), len);
        readPos_ += len;
        return true;
    }
    
    // 고정 길이 문자열 읽기
    bool ReadFixedString(char* outStr, size_t maxLen) {
        if (error_ || readPos_ + maxLen > bufferSize_) {
            error_ = true;
            return false;
        }
        
        memcpy(outStr, &buffer_[readPos_], maxLen);
        readPos_ += maxLen;
        return true;
    }
    
    // 바이트 배열 읽기
    bool ReadBytes(BYTE* outData, size_t size) {
        if (error_ || readPos_ + size > bufferSize_) {
            error_ = true;
            return false;
        }
        
        memcpy(outData, &buffer_[readPos_], size);
        readPos_ += size;
        return true;
    }
    
    size_t GetRemainSize() const {
        return bufferSize_ - readPos_;
    }
    
    bool IsEnd() const {
        return readPos_ >= bufferSize_;
    }
    
    bool HasError() const {
        return error_;
    }
};
```

```cpp
void Session::OnPacket(const PacketHeader& header) {
    // 패킷 크기 검증
    if (!IsValidPacketSize(header.size)) {
        std::cout << "[Session] 잘못된 패킷 크기: " 
                 << header.size << std::endl;
        Disconnect();
        return;
    }
    
    // 패킷 ID 검증
    if (!IsValidPacketID(header.packetId)) {
        std::cout << "[Session] 잘못된 패킷 ID: " 
                 << header.packetId << std::endl;
        Disconnect();
        return;
    }
    
    // 패킷 데이터 추출
    std::vector<BYTE> packetData(header.size);
    recvBuffer_.Peek(packetData.data(), header.size);
    
    // 패킷 리더 생성
    PacketReader reader(packetData.data(), header.size);
    
    // 패킷 타입에 따라 처리
    PacketID packetId = static_cast<PacketID>(header.packetId);
    
    switch (packetId) {
        case PacketID::ECHO_REQ:
        {
            EchoPacketHandler handler;
            handler.Handle(this, reader);
            break;
        }
        case PacketID::LOGIN_REQ:
        {
            LoginPacketHandler handler;
            handler.Handle(this, reader);
            break;
        }
        case PacketID::CHAT_REQ:
        {
            ChatPacketHandler handler;
            handler.Handle(this, reader);
            break;
        }
        default:
            std::cout << "[Session] 알 수 없는 패킷: " 
                     << header.packetId << std::endl;
            break;
    }
}
```

나머지 파일들(ServerGlobal, SessionPool, AcceptThread, WorkerThread, Main 등)은 이전 챕터와 동일하다.
