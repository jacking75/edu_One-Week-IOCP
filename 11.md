# Chapter 11. 룸(Room) 기반 게임 서버
대부분의 멀티플레이 게임은 플레이어들을 작은 그룹으로 나누어 관리한다. 배틀그라운드의 매치, 리그 오브 레전드의 게임 방, 포커 게임의 테이블이 모두 룸(Room) 개념이다. 룸은 플레이어들을 논리적으로 격리하여 상호작용을 제한하고, 게임 상태를 독립적으로 관리한다. 이번 챕터에서는 룸 기반 아키텍처를 구현하여 확장 가능하고 안정적인 멀티플레이 게임 서버를 완성한다.

## 11.1 룸(Room) 개념과 설계

**룸이 필요한 이유**

서버에 수만 명의 플레이어가 접속했을 때, 모든 플레이어가 서로 상호작용한다면 네트워크와 CPU가 감당할 수 없다. 한 플레이어가 움직일 때마다 수만 개의 패킷을 브로드캐스트해야 한다면 대역폭이 기하급수적으로 증가한다. 룸은 이 문제를 해결한다.

```
전역 공간 vs 룸 기반:

[전역 공간]
Player A의 행동 → 전체 9,999명에게 브로드캐스트
네트워크: O(N²) - 폭발적 증가

[룸 기반]
Room 1 (4명): Player A → 3명에게만 브로드캐스트
Room 2 (4명): Player B → 3명에게만 브로드캐스트
네트워크: O(R * M²) - R=룸 수, M=룸당 평균 인원
```

룸은 게임 로직도 단순화한다. 체스 게임이라면 각 체스판(룸)은 독립적인 게임 상태를 가지며, 다른 체스판과 영향을 주고받지 않는다. 이는 동기화 문제를 크게 줄인다.

**룸의 생명주기**

룸은 생성, 대기, 진행, 종료의 생명주기를 가진다.

```
Room Lifecycle:

┌─────────┐    플레이어      ┌─────────┐    게임      ┌─────────┐
│ Created │───입장 완료──→│ Waiting │───시작──→│ Playing │
└─────────┘              └─────────┘          └─────────┘
                                                    │
                                                게임 종료
                                                    ▼
                                              ┌──────────┐
                                              │ Finished │
                                              └──────────┘
                                                    │
                                               타이머 만료
                                                    ▼
                                              [메모리 해제]
```

생성된 룸은 플레이어들이 입장할 때까지 대기 상태다. 충분한 인원이 모이거나 방장이 시작 버튼을 누르면 게임이 시작된다. 게임이 끝나면 일정 시간 후 자동으로 룸이 삭제된다.

**룸의 스레드 모델**

10장에서 로직 스레드를 분리했다. 룸 기반 서버에서는 각 룸이 독립적인 작업 큐를 가지는 것이 이상적이다. 하지만 룸마다 스레드를 할당하면 수천 개의 룸에 수천 개의 스레드가 필요하므로 비현실적이다.

실용적인 접근은 로직 스레드 풀을 공유하되, 같은 룸의 작업은 순차적으로 처리되도록 보장하는 것이다. 룸 ID를 해시하여 특정 로직 스레드에 할당하거나, 룸별로 작업 큐를 두고 로직 스레드가 여러 룸의 큐를 순회하며 처리한다.

```
Room-Thread Mapping:

┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
│  Room 1  │  │  Room 2  │  │  Room 3  │  │  Room 4  │
│ [Queue]  │  │ [Queue]  │  │ [Queue]  │  │ [Queue]  │
└────┬─────┘  └────┬─────┘  └────┬─────┘  └────┬─────┘
     │             │             │             │
     └──────┬──────┴──────┬──────┴──────┬──────┘
            │             │             │
      ┌─────▼────┐  ┌─────▼────┐  ┌─────▼────┐
      │ Logic    │  │ Logic    │  │ Logic    │
      │ Thread 1 │  │ Thread 2 │  │ Thread 3 │
      └──────────┘  └──────────┘  └──────────┘

→ 룸은 해시로 특정 스레드에 매핑
→ 같은 룸의 작업은 항상 같은 스레드에서 처리
```

이 방식은 구현이 간단하고 성능도 우수하다. 룸 수가 스레드 수보다 많으면 자동으로 부하가 분산된다.

**룸 데이터 구조**

룸은 참가자 목록, 게임 상태, 설정 정보를 포함한다.

```cpp
enum class RoomState {
    Waiting,    // 플레이어 대기 중
    Playing,    // 게임 진행 중
    Finished    // 게임 종료
};

struct RoomConfig {
    uint32_t maxPlayers;      // 최대 인원
    uint32_t minPlayers;      // 최소 시작 인원
    bool isPrivate;           // 비공개 방 여부
    std::string password;     // 비밀번호 (비공개 방)
};

class Room {
private:
    uint64_t roomId_;
    std::string roomName_;
    RoomState state_;
    RoomConfig config_;
    
    // 참가자 관리
    std::unordered_map<uint64_t, std::shared_ptr<Session>> players_;
    std::mutex playersMutex_;
    
    uint64_t hostSessionId_;  // 방장
    
    // 게임 로직 스레드 할당
    int assignedThreadId_;
    
    // 작업 큐
    std::unique_ptr<JobQueue> pJobQueue_;
    
public:
    Room(uint64_t id, const std::string& name, const RoomConfig& config);
    
    bool CanJoin() const;
    bool AddPlayer(std::shared_ptr<Session> pSession);
    void RemovePlayer(uint64_t sessionId);
    void BroadcastPacket(const char* pData, int size, uint64_t exceptSessionId = 0);
    
    void StartGame();
    void EndGame();
};
```

룸은 세션의 `shared_ptr`을 보관하여 플레이어가 연결이 끊겨도 안전하게 참조할 수 있다. 뮤텍스는 I/O 스레드와 로직 스레드가 동시에 플레이어 목록을 수정할 때를 대비한다.
  

## 11.2 룸 매니저 구현

**RoomManager의 역할**

RoomManager는 모든 룸을 중앙에서 관리한다. 룸 생성, 삭제, 조회, 검색 기능을 제공한다.

```cpp
class RoomManager {
private:
    // 룸 ID → Room 객체
    std::unordered_map<uint64_t, std::shared_ptr<Room>> rooms_;
    std::shared_mutex roomsMutex_;  // C++17 shared_mutex
    
    std::atomic<uint64_t> nextRoomId_;
    
    // 로직 스레드 풀
    std::vector<std::unique_ptr<LogicWorker>> logicWorkers_;
    int logicThreadCount_;

public:
    RoomManager(int logicThreadCount)
        : nextRoomId_(1)
        , logicThreadCount_(logicThreadCount) {
        
        // 로직 워커 초기화는 별도로 수행
    }
    
    std::shared_ptr<Room> CreateRoom(const std::string& name, const RoomConfig& config);
    bool DestroyRoom(uint64_t roomId);
    std::shared_ptr<Room> FindRoom(uint64_t roomId);
    std::vector<std::shared_ptr<Room>> GetAvailableRooms();
    
    int GetRoomCount() const;
    int GetTotalPlayerCount() const;
};
```

`shared_mutex`를 사용하면 여러 스레드가 동시에 읽기를 수행할 수 있어 성능이 향상된다. 룸 조회는 빈번하지만 생성/삭제는 드물기 때문에 적합하다.

**룸 생성과 스레드 할당**

새 룸을 생성할 때 로직 스레드를 할당한다. 간단한 라운드 로빈이나 해시 기반 할당을 사용한다.

```cpp
std::shared_ptr<Room> RoomManager::CreateRoom(const std::string& name, 
                                               const RoomConfig& config) {
    uint64_t roomId = nextRoomId_.fetch_add(1, std::memory_order_relaxed);
    
    // 로직 스레드 할당 (라운드 로빈)
    int threadId = roomId % logicThreadCount_;
    
    auto pRoom = std::make_shared<Room>(roomId, name, config, threadId);
    
    {
        std::unique_lock<std::shared_mutex> lock(roomsMutex_);
        rooms_[roomId] = pRoom;
    }
    
    std::cout << "[RoomManager] Room created: ID=" << roomId 
              << ", Name=" << name 
              << ", Thread=" << threadId << "\n";
    
    return pRoom;
}
```

라운드 로빈 방식은 단순하지만 효과적이다. 룸이 많아지면 자동으로 부하가 분산된다. 더 정교한 방법은 각 스레드의 현재 부하를 측정하여 가장 여유 있는 스레드에 할당하는 것이다.

**룸 조회와 검색**

플레이어가 참가할 룸을 찾을 때는 여러 조건으로 필터링한다.

```cpp
std::vector<std::shared_ptr<Room>> RoomManager::GetAvailableRooms() {
    std::vector<std::shared_ptr<Room>> availableRooms;
    
    std::shared_lock<std::shared_mutex> lock(roomsMutex_);
    
    for (const auto& pair : rooms_) {
        auto& pRoom = pair.second;
        
        // 참가 가능한 방만 선택
        if (pRoom->GetState() == RoomState::Waiting && pRoom->CanJoin()) {
            availableRooms.push_back(pRoom);
        }
    }
    
    return availableRooms;
}

std::shared_ptr<Room> RoomManager::FindRoom(uint64_t roomId) {
    std::shared_lock<std::shared_mutex> lock(roomsMutex_);
    
    auto it = rooms_.find(roomId);
    if (it != rooms_.end()) {
        return it->second;
    }
    
    return nullptr;
}
```

`shared_lock`을 사용하여 여러 스레드가 동시에 읽을 수 있다. 게임 서버에서 룸 목록 조회는 매우 빈번하므로 이런 최적화가 중요하다.

**룸 자동 삭제**

비어있거나 게임이 끝난 룸은 일정 시간 후 자동 삭제되어야 한다. 타이머를 활용한다.

```cpp
void Room::ScheduleAutoDestroy(TimerManager* pTimerManager, RoomManager* pRoomManager) {
    // 5분 후 자동 삭제
    pTimerManager->AddTimer(
        std::chrono::minutes(5),
        [roomId = this->roomId_, pRoomManager]() {
            auto pRoom = pRoomManager->FindRoom(roomId);
            if (pRoom && pRoom->GetState() == RoomState::Finished) {
                pRoomManager->DestroyRoom(roomId);
            }
        }
    );
}

bool RoomManager::DestroyRoom(uint64_t roomId) {
    std::shared_ptr<Room> pRoom;
    
    {
        std::unique_lock<std::shared_mutex> lock(roomsMutex_);
        auto it = rooms_.find(roomId);
        
        if (it == rooms_.end()) {
            return false;
        }
        
        pRoom = it->second;
        rooms_.erase(it);
    }
    
    // 룸에 남은 플레이어 강제 퇴장
    pRoom->KickAllPlayers();
    
    std::cout << "[RoomManager] Room destroyed: ID=" << roomId << "\n";
    return true;
}
```
  

## 11.3 룸 입장/퇴장 처리

**룸 입장 프로세스**

플레이어가 룸에 입장할 때는 여러 검증을 거친다.

```cpp
bool Room::AddPlayer(std::shared_ptr<Session> pSession) {
    std::lock_guard<std::mutex> lock(playersMutex_);
    
    // 입장 가능 여부 확인
    if (state_ != RoomState::Waiting) {
        std::cout << "[Room " << roomId_ << "] Cannot join: game already started\n";
        return false;
    }
    
    if (players_.size() >= config_.maxPlayers) {
        std::cout << "[Room " << roomId_ << "] Cannot join: room full\n";
        return false;
    }
    
    uint64_t sessionId = pSession->GetSessionId();
    
    if (players_.count(sessionId) > 0) {
        std::cout << "[Room " << roomId_ << "] Player already in room\n";
        return false;
    }
    
    // 플레이어 추가
    players_[sessionId] = pSession;
    
    // 첫 입장자를 방장으로
    if (players_.size() == 1) {
        hostSessionId_ = sessionId;
    }
    
    std::cout << "[Room " << roomId_ << "] Player " << sessionId 
              << " joined (" << players_.size() << "/" 
              << config_.maxPlayers << ")\n";
    
    // 기존 플레이어들에게 새 플레이어 알림
    NotifyPlayerJoined(sessionId);
    
    // 새 플레이어에게 현재 룸 상태 전송
    SendRoomStateToPlayer(pSession);
    
    return true;
}
```

입장에 성공하면 기존 플레이어들에게 알리고, 새 플레이어에게는 현재 룸의 상태를 전송한다.

**룸 상태 동기화**

새로 입장한 플레이어는 누가 이미 방에 있는지, 방 설정은 어떤지 알아야 한다.

```cpp
void Room::SendRoomStateToPlayer(std::shared_ptr<Session> pSession) {
    RoomStatePacket packet;
    packet.header.packetId = PacketId::RoomState;
    packet.roomId = roomId_;
    packet.state = state_;
    packet.playerCount = players_.size();
    packet.maxPlayers = config_.maxPlayers;
    packet.hostSessionId = hostSessionId_;
    
    // 플레이어 목록 (간단히 ID만)
    int index = 0;
    for (const auto& pair : players_) {
        if (index >= 16) break;  // 최대 16명
        packet.playerIds[index++] = pair.first;
    }
    
    packet.header.size = sizeof(RoomStatePacket);
    
    pSession->Send(reinterpret_cast<char*>(&packet), sizeof(packet));
}
```

**플레이어 퇴장 처리**

플레이어가 퇴장하거나 연결이 끊기면 룸에서 제거한다.

```cpp
void Room::RemovePlayer(uint64_t sessionId) {
    std::lock_guard<std::mutex> lock(playersMutex_);
    
    auto it = players_.find(sessionId);
    if (it == players_.end()) {
        return;
    }
    
    players_.erase(it);
    
    std::cout << "[Room " << roomId_ << "] Player " << sessionId 
              << " left (" << players_.size() << " remaining)\n";
    
    // 방장이 나갔으면 다른 사람을 방장으로
    if (sessionId == hostSessionId_ && !players_.empty()) {
        hostSessionId_ = players_.begin()->first;
        std::cout << "[Room " << roomId_ << "] New host: " << hostSessionId_ << "\n";
    }
    
    // 나머지 플레이어들에게 알림
    NotifyPlayerLeft(sessionId);
    
    // 방이 비었으면 종료 예약
    if (players_.empty()) {
        std::cout << "[Room " << roomId_ << "] Room is empty. Scheduling destroy.\n";
        // 타이머로 자동 삭제 예약
    }
}
```

방장이 나가면 자동으로 다음 사람이 방장이 된다. 방이 완전히 비면 일정 시간 후 삭제된다.

**입장/퇴장 알림 패킷**

룸의 플레이어 구성이 바뀌면 모든 플레이어에게 알린다.

```cpp
void Room::NotifyPlayerJoined(uint64_t sessionId) {
    PlayerJoinedPacket packet;
    packet.header.packetId = PacketId::PlayerJoined;
    packet.header.size = sizeof(PlayerJoinedPacket);
    packet.roomId = roomId_;
    packet.sessionId = sessionId;
    packet.currentPlayerCount = players_.size();
    
    // 입장한 플레이어 제외하고 브로드캐스트
    BroadcastPacket(reinterpret_cast<char*>(&packet), sizeof(packet), sessionId);
}

void Room::NotifyPlayerLeft(uint64_t sessionId) {
    PlayerLeftPacket packet;
    packet.header.packetId = PacketId::PlayerLeft;
    packet.header.size = sizeof(PlayerLeftPacket);
    packet.roomId = roomId_;
    packet.sessionId = sessionId;
    packet.currentPlayerCount = players_.size();
    
    BroadcastPacket(reinterpret_cast<char*>(&packet), sizeof(packet));
}
```
  

## 11.4 룸 내 브로드캐스트

**기본 브로드캐스트 구현**

룸의 핵심 기능은 한 플레이어의 행동을 다른 플레이어들에게 전파하는 것이다.

```cpp
void Room::BroadcastPacket(const char* pData, int size, uint64_t exceptSessionId) {
    std::lock_guard<std::mutex> lock(playersMutex_);
    
    for (const auto& pair : players_) {
        if (pair.first == exceptSessionId) {
            continue;  // 자신에게는 보내지 않음
        }
        
        auto& pSession = pair.second;
        pSession->Send(pData, size);
    }
}
```

이 방식은 단순하지만 패킷을 플레이어 수만큼 복사한다. 10명이면 10번 복사되므로 비효율적이다.

**Zero-Copy 브로드캐스트**

8장에서 다룬 Zero-Copy 기법을 활용하여 패킷 데이터를 공유한다.

```cpp
class SharedPacketBuffer {
private:
    std::shared_ptr<std::vector<char>> pData_;
    
public:
    SharedPacketBuffer(const char* pData, int size) {
        pData_ = std::make_shared<std::vector<char>>(pData, pData + size);
    }
    
    const char* GetData() const { return pData_->data(); }
    int GetSize() const { return pData_->size(); }
    std::shared_ptr<std::vector<char>> GetSharedPtr() { return pData_; }
};

void Room::BroadcastPacketOptimized(const char* pData, int size, 
                                    uint64_t exceptSessionId) {
    // 패킷을 한 번만 공유 메모리로 복사
    auto sharedBuffer = std::make_shared<std::vector<char>>(pData, pData + size);
    
    std::lock_guard<std::mutex> lock(playersMutex_);
    
    for (const auto& pair : players_) {
        if (pair.first == exceptSessionId) {
            continue;
        }
        
        auto& pSession = pair.second;
        pSession->SendShared(sharedBuffer);
    }
}
```

`SendShared`는 `shared_ptr`를 송신 큐에 넣어 참조 카운팅으로 메모리를 관리한다. 실제 데이터는 한 번만 복사되고, 모든 세션이 같은 메모리를 참조한다.

**선택적 브로드캐스트**

때로는 특정 조건을 만족하는 플레이어에게만 패킷을 보내야 한다.

```cpp
void Room::BroadcastPacketIf(const char* pData, int size,
                              std::function<bool(const std::shared_ptr<Session>&)> predicate) {
    std::lock_guard<std::mutex> lock(playersMutex_);
    
    for (const auto& pair : players_) {
        auto& pSession = pair.second;
        
        if (predicate(pSession)) {
            pSession->Send(pData, size);
        }
    }
}

// 사용 예: 특정 팀에만 전송
room->BroadcastPacketIf(packet, size, [](const auto& pSession) {
    return pSession->GetTeam() == TeamType::Red;
});
```

**브로드캐스트 최적화: 배치 전송**

여러 패킷을 하나씩 브로드캐스트하는 대신, 모아서 한 번에 보내면 효율적이다.

```cpp
class BroadcastBatcher {
private:
    std::vector<char> batchBuffer_;
    Room* pRoom_;
    
public:
    BroadcastBatcher(Room* pRoom) : pRoom_(pRoom) {
        batchBuffer_.reserve(4096);
    }
    
    void AddPacket(const char* pData, int size) {
        batchBuffer_.insert(batchBuffer_.end(), pData, pData + size);
    }
    
    void Flush(uint64_t exceptSessionId = 0) {
        if (!batchBuffer_.empty()) {
            pRoom_->BroadcastPacket(batchBuffer_.data(), 
                                   batchBuffer_.size(), 
                                   exceptSessionId);
            batchBuffer_.clear();
        }
    }
    
    ~BroadcastBatcher() {
        Flush();
    }
};

// 사용 예
void Room::UpdateGameState() {
    BroadcastBatcher batcher(this);
    
    // 여러 패킷 추가
    batcher.AddPacket(packet1, size1);
    batcher.AddPacket(packet2, size2);
    batcher.AddPacket(packet3, size3);
    
    // 소멸자에서 자동으로 Flush
}
```

이 방식은 네트워크 호출 횟수를 줄여 오버헤드를 감소시킨다.

```cpp
           players_.size() < config_.maxPlayers;

bool Room::AddPlayer(std::shared_ptr<Session> pSession) {
    std::lock_guard<std::mutex> lock(playersMutex_);
    
    if (state_.load(std::memory_order_acquire) != RoomState::Waiting) {
        return false;
    }
    
    if (players_.size() >= config_.maxPlayers) {
        return false;
    }
    
    uint64_t sessionId = pSession->GetSessionId();
    
    if (players_.count(sessionId) > 0) {
        return false;
    }
    
    players_[sessionId] = pSession;
    
    if (players_.size() == 1) {
        hostSessionId_ = sessionId;
    }
    
    std::cout << "[Room " << roomId_ << "] Player " << sessionId 
              << " joined (" << players_.size() << "/" 
              << config_.maxPlayers << ")\n";
    
    NotifyPlayerJoined(sessionId);
    SendRoomStateToPlayer(pSession);
    
    return true;
}

void Room::RemovePlayer(uint64_t sessionId) {
    std::lock_guard<std::mutex> lock(playersMutex_);
    
    auto it = players_.find(sessionId);
    if (it == players_.end()) {
        return;
    }
    
    players_.erase(it);
    
    std::cout << "[Room " << roomId_ << "] Player " << sessionId 
              << " left (" << players_.size() << " remaining)\n";
    
    if (sessionId == hostSessionId_ && !players_.empty()) {
        hostSessionId_ = players_.begin()->first;
        std::cout << "[Room " << roomId_ << "] New host: " << hostSessionId_ << "\n";
    }
    
    NotifyPlayerLeft(sessionId);
    
    if (players_.empty()) {
        std::cout << "[Room " << roomId_ << "] Room is empty.\n";
    }

void Room::KickAllPlayers() {
    std::lock_guard<std::mutex> lock(playersMutex_);
    
    PlayerKickedPacket packet;
    packet.header.packetId = PacketId::PlayerKicked;
    packet.header.size = sizeof(PlayerKickedPacket);
    packet.reason = KickReason::RoomClosed;
    
    for (auto& pair : players_) {
        auto& pSession = pair.second;
        pSession->Send(reinterpret_cast<char*>(&packet), sizeof(packet));
    }
    
    players_.clear();
}

void Room::BroadcastPacket(const char* pData, int size, uint64_t exceptSessionId) {
    std::lock_guard<std::mutex> lock(playersMutex_);
    
    for (const auto& pair : players_) {
        if (pair.first == exceptSessionId) {
            continue;
        }
        
        auto& pSession = pair.second;
        pSession->Send(pData, size);
    }
}
```

```cpp
std::shared_ptr<Room> RoomManager::CreateRoom(const std::string& name, 
                                               const RoomConfig& config) {
    uint64_t roomId = nextRoomId_.fetch_add(1, std::memory_order_relaxed);

        rooms_[roomId] = pRoom;

    std::shared_ptr<Room> pRoom;
    
    {
        std::unique_lock<std::shared_mutex> lock(roomsMutex_);
        auto it = rooms_.find(roomId);
        
        if (it == rooms_.end()) {
            return false;
        }
        
        pRoom = it->second;
        rooms_.erase(it);

        return it->second;

    return nullptr;

std::vector<std::shared_ptr<Room>> RoomManager::GetAvailableRooms() {
    std::vector<std::shared_ptr<Room>> availableRooms;
    
    std::shared_lock<std::shared_mutex> lock(roomsMutex_);
    
    for (const auto& pair : rooms_) {
        auto& pRoom = pair.second;
        
        if (pRoom->GetState() == RoomState::Waiting && pRoom->CanJoin()) {
            availableRooms.push_back(pRoom);
        }
```

```cpp
#include "GameServer.h"
#include "RoomManager.h"
#include <iostream>

class GameServer {
private:
    // 기존 멤버들...
    std::unique_ptr<RoomManager> pRoomManager_;
    
public:
    bool Initialize(int logicThreadCount = 4) {
        // 기존 초기화...
        
        // 룸 매니저 생성
        pRoomManager_ = std::make_unique<RoomManager>(logicThreadCount);
        
        std::cout << "GameServer initialized with Room system.\n";
        return true;
    }
    
    void HandleCreateRoomPacket(std::shared_ptr<Session> pSession, 
                               const CreateRoomPacket* pPacket) {
        RoomConfig config;
        config.maxPlayers = pPacket->maxPlayers;
        config.minPlayers = pPacket->minPlayers;
        config.isPrivate = pPacket->isPrivate;
        config.password = pPacket->password;
        
        auto pRoom = pRoomManager_->CreateRoom(pPacket->roomName, config);
        
        CreateRoomResponsePacket response;
        response.header.packetId = PacketId::CreateRoomResponse;
        response.header.size = sizeof(CreateRoomResponsePacket);
        response.success = (pRoom != nullptr);
        response.roomId = pRoom ? pRoom->GetRoomId() : 0;
        
        pSession->Send(reinterpret_cast<char*>(&response), sizeof(response));
        
        // 자동으로 방 입장
        if (pRoom) {
            pRoom->AddPlayer(pSession);
            pSession->SetCurrentRoom(pRoom);
        }
    }
    
    void HandleJoinRoomPacket(std::shared_ptr<Session> pSession, 
                             const JoinRoomPacket* pPacket) {
        auto pRoom = pRoomManager_->FindRoom(pPacket->roomId);
        
        JoinRoomResponsePacket response;
        response.header.packetId = PacketId::JoinRoomResponse;
        response.header.size = sizeof(JoinRoomResponsePacket);
        
        if (!pRoom) {
            response.success = false;
            pSession->Send(reinterpret_cast<char*>(&response), sizeof(response));
            return;
        }
        
        bool joined = pRoom->AddPlayer(pSession);
        response.success = joined;
        response.roomId = pPacket->roomId;
        
        pSession->Send(reinterpret_cast<char*>(&response), sizeof(response));
        
        if (joined) {
            pSession->SetCurrentRoom(pRoom);
        }
    }
    
    void MonitorRooms() {
        while (isRunning_) {
            std::this_thread::sleep_for(std::chrono::seconds(30));
            pRoomManager_->PrintStatistics();
        }
    }
};
```

이 코드는 룸 기반 게임 서버의 완전한 구현을 보여준다. RoomManager가 모든 룸을 관리하고, 각 Room은 독립적으로 플레이어와 게임 상태를 관리한다. 브로드캐스트 최적화와 상태 동기화를 통해 효율적인 멀티플레이 환경을 제공한다.
