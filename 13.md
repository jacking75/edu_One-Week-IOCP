# Chapter 13. 오목 게임 클라이언트 구현
이 장에서는 Chapter 12에서 구현한 오목 게임 서버와 통신하는 Win32 API 기반 클라이언트를 구현한다. 게임 서버 학습서이지만 완성된 게임을 테스트하고 동작을 확인하기 위해서는 클라이언트가 필요하다. 클라이언트는 최대한 간결하게 구현하되, 실제 게임을 플레이할 수 있는 수준의 GUI와 네트워크 기능을 모두 갖추도록 한다.

Win32 API의 GDI(Graphics Device Interface)를 사용하여 오목판을 그리고, 마우스 클릭으로 돌을 놓으며, 별도의 네트워크 스레드에서 서버와 통신한다. 클라이언트 아키텍처는 UI 스레드와 네트워크 스레드로 분리하여 서버 응답을 기다리는 동안에도 화면이 멈추지 않도록 구성한다.

```
┌─────────────────────────────────────────┐
│         Win32 Client Application        │
├─────────────────────────────────────────┤
│  ┌───────────────┐  ┌───────────────┐  │
│  │  UI Thread    │  │ Network Thread│  │
│  │               │  │               │  │
│  │ - WndProc     │  │ - Connect     │  │
│  │ - GDI Draw    │  │ - Send/Recv   │  │
│  │ - Mouse Input │  │ - Packet      │  │
│  │               │  │   Processing  │  │
│  └───────┬───────┘  └───────┬───────┘  │
│          │                  │          │
│          └──────┬───────────┘          │
│                 │                      │
│         ┌───────▼────────┐             │
│         │  Game State    │             │
│         │  (Thread-Safe) │             │
│         └────────────────┘             │
└─────────────────────────────────────────┘
```

클라이언트는 네트워크 프로그래밍보다는 UI와 게임 로직 표현에 집중하므로, Winsock의 기본적인 블로킹 소켓이나 간단한 비동기 방식을 사용한다. IOCP는 서버 측 기술이므로 클라이언트에서는 사용하지 않는다.

## 13.1 Win32 윈도우 생성과 메시지 루프
Win32 애플리케이션은 윈도우 클래스를 등록하고, 윈도우를 생성한 뒤, 메시지 루프를 돌며 사용자 입력과 시스템 이벤트를 처리한다. 오목 게임 클라이언트의 윈도우는 19x19 오목판을 표시할 수 있는 크기로 생성하며, 타이틀 바에 현재 게임 상태(대기 중, 게임 중, 승리/패배 등)를 표시한다.

**윈도우 클래스 등록 및 생성**

Win32 애플리케이션의 진입점은 `WinMain` 함수다. 먼저 `WNDCLASSEX` 구조체를 채워 윈도우 클래스를 등록하고, `CreateWindowEx` 함수로 윈도우를 생성한다.

```cpp
LRESULT CALLBACK WndProc(HWND hwnd, UINT msg, WPARAM wParam, LPARAM lParam);

int WINAPI WinMain(HINSTANCE hInstance, HINSTANCE hPrevInstance, 
                   LPSTR lpCmdLine, int nCmdShow)
{
    WNDCLASSEX wc = {};
    wc.cbSize = sizeof(WNDCLASSEX);
    wc.style = CS_HREDRAW | CS_VREDRAW;
    wc.lpfnWndProc = WndProc;
    wc.hInstance = hInstance;
    wc.hCursor = LoadCursor(nullptr, IDC_ARROW);
    wc.hbrBackground = (HBRUSH)(COLOR_WINDOW + 1);
    wc.lpszClassName = L"OmokClientClass";

    if (!RegisterClassEx(&wc)) {
        MessageBox(nullptr, L"윈도우 클래스 등록 실패", L"Error", MB_OK);
        return 0;
    }

    constexpr int BOARD_SIZE = 19;
    constexpr int CELL_SIZE = 30;
    constexpr int MARGIN = 40;
    int windowWidth = CELL_SIZE * (BOARD_SIZE - 1) + MARGIN * 2;
    int windowHeight = CELL_SIZE * (BOARD_SIZE - 1) + MARGIN * 2 + 100;

    HWND hwnd = CreateWindowEx(
        0, L"OmokClientClass", L"오목 게임 클라이언트",
        WS_OVERLAPPEDWINDOW, CW_USEDEFAULT, CW_USEDEFAULT,
        windowWidth, windowHeight, nullptr, nullptr, hInstance, nullptr);

    if (!hwnd) {
        MessageBox(nullptr, L"윈도우 생성 실패", L"Error", MB_OK);
        return 0;
    }

    ShowWindow(hwnd, nCmdShow);
    UpdateWindow(hwnd);

    // 메시지 루프
    MSG msg;
    while (GetMessage(&msg, nullptr, 0, 0)) {
        TranslateMessage(&msg);
        DispatchMessage(&msg);
    }

    return static_cast<int>(msg.wParam);
}
```

윈도우 크기는 오목판의 셀 크기와 여백을 고려하여 계산한다. 19x19 오목판을 30픽셀 간격으로 그리고, 좌우/상하에 40픽셀의 여백을 둔다. 하단에는 게임 상태를 표시할 공간으로 100픽셀을 추가한다.

**메시지 처리 함수**

`WndProc` 함수는 윈도우로 전달되는 모든 메시지를 처리한다. 주요 메시지로는 `WM_CREATE`(윈도우 생성 시 초기화), `WM_PAINT`(화면 그리기), `WM_LBUTTONDOWN`(마우스 클릭), `WM_DESTROY`(종료 처리) 등이 있다.

```cpp
LRESULT CALLBACK WndProc(HWND hwnd, UINT msg, WPARAM wParam, LPARAM lParam)
{
    switch (msg) {
    case WM_CREATE:
        OnCreate(hwnd);
        return 0;
    case WM_PAINT:
        OnPaint(hwnd);
        return 0;
    case WM_LBUTTONDOWN:
        OnLButtonDown(hwnd, LOWORD(lParam), HIWORD(lParam));
        return 0;
    case WM_DESTROY:
        OnDestroy(hwnd);
        PostQuitMessage(0);
        return 0;
    }
    return DefWindowProc(hwnd, msg, wParam, lParam);
}
```

각 메시지에 대한 처리는 별도의 함수로 분리하여 코드 가독성을 높인다. 이후 섹션에서 각 함수의 구현을 자세히 다룬다.

**전역 상태 관리**

클라이언트의 게임 상태는 전역 변수 또는 싱글톤 패턴으로 관리한다. 네트워크 스레드와 UI 스레드가 동시에 접근할 수 있으므로 크리티컬 섹션이나 뮤텍스로 보호해야 한다.

```cpp
struct GameState {
    std::mutex mtx;
    int board[19][19] = {};  // 0: 빈칸, 1: 흑돌, 2: 백돌
    int myStone = 0;         // 내가 조종하는 돌 (1 or 2)
    int currentTurn = 1;     // 현재 턴 (1: 흑, 2: 백)
    bool isGameStarted = false;
    bool isGameOver = false;
    std::wstring statusMessage = L"서버 연결 대기 중...";
};

GameState g_gameState;
```

이 구조체는 오목판의 상태, 현재 턴, 게임 시작/종료 여부, 화면에 표시할 메시지 등을 담는다. UI 스레드는 이 상태를 읽어 화면을 그리고, 네트워크 스레드는 서버로부터 받은 패킷을 파싱하여 이 상태를 업데이트한다.

## 13.2 GDI를 이용한 오목판 렌더링
GDI는 Windows의 2D 그래픽 API로, 선, 도형, 텍스트 등을 그릴 수 있다. 오목판은 19x19 격자로 이루어져 있으며, 각 교차점에 돌을 그린다. GDI의 `LineTo`, `Ellipse`, `TextOut` 등의 함수를 사용하여 오목판과 돌, 상태 텍스트를 렌더링한다.

**더블 버퍼링**

화면 깜빡임을 방지하기 위해 더블 버퍼링 기법을 사용한다. 메모리 DC(Device Context)에 먼저 그린 후, 한 번에 화면 DC로 복사한다.

```cpp
void OnPaint(HWND hwnd)
{
    PAINTSTRUCT ps;
    HDC hdc = BeginPaint(hwnd, &ps);

    // 클라이언트 영역 크기 얻기
    RECT clientRect;
    GetClientRect(hwnd, &clientRect);

    // 메모리 DC 생성 (더블 버퍼링)
    HDC memDC = CreateCompatibleDC(hdc);
    HBITMAP memBitmap = CreateCompatibleBitmap(hdc, 
        clientRect.right, clientRect.bottom);
    HBITMAP oldBitmap = (HBITMAP)SelectObject(memDC, memBitmap);

    // 배경 칠하기
    HBRUSH bgBrush = CreateSolidBrush(RGB(220, 179, 92));
    FillRect(memDC, &clientRect, bgBrush);
    DeleteObject(bgBrush);

    // 오목판 그리기
    DrawBoard(memDC);

    // 돌 그리기
    DrawStones(memDC);

    // 상태 텍스트 그리기
    DrawStatus(memDC, clientRect);

    // 메모리 DC 내용을 화면 DC로 복사
    BitBlt(hdc, 0, 0, clientRect.right, clientRect.bottom, 
           memDC, 0, 0, SRCCOPY);

    // 정리
    SelectObject(memDC, oldBitmap);
    DeleteObject(memBitmap);
    DeleteDC(memDC);

    EndPaint(hwnd, &ps);
}
```

**오목판 격자 그리기**

19x19 격자를 그리기 위해 가로선 19개, 세로선 19개를 그린다. 특정 좌표(천원, 화점 등)에는 작은 원을 그려 표시한다.

```cpp
void DrawBoard(HDC hdc)
{
    constexpr int BOARD_SIZE = 19;
    constexpr int CELL_SIZE = 30;
    constexpr int MARGIN = 40;

    HPEN pen = CreatePen(PS_SOLID, 1, RGB(0, 0, 0));
    HPEN oldPen = (HPEN)SelectObject(hdc, pen);

    // 가로선
    for (int i = 0; i < BOARD_SIZE; ++i) {
        int y = MARGIN + i * CELL_SIZE;
        MoveToEx(hdc, MARGIN, y, nullptr);
        LineTo(hdc, MARGIN + (BOARD_SIZE - 1) * CELL_SIZE, y);
    }

    // 세로선
    for (int i = 0; i < BOARD_SIZE; ++i) {
        int x = MARGIN + i * CELL_SIZE;
        MoveToEx(hdc, x, MARGIN, nullptr);
        LineTo(hdc, x, MARGIN + (BOARD_SIZE - 1) * CELL_SIZE);
    }

    // 화점 표시 (3, 9, 15 교차점)
    HBRUSH dotBrush = CreateSolidBrush(RGB(0, 0, 0));
    HBRUSH oldBrush = (HBRUSH)SelectObject(hdc, dotBrush);
    
    int starPoints[] = { 3, 9, 15 };
    for (int i : starPoints) {
        for (int j : starPoints) {
            int x = MARGIN + j * CELL_SIZE;
            int y = MARGIN + i * CELL_SIZE;
            Ellipse(hdc, x - 3, y - 3, x + 3, y + 3);
        }
    }

    SelectObject(hdc, oldBrush);
    DeleteObject(dotBrush);
    SelectObject(hdc, oldPen);
    DeleteObject(pen);
}
```

**돌 렌더링**

돌은 원으로 표현하며, 흑돌은 검은색, 백돌은 흰색으로 채운다. 게임 상태 배열을 순회하면서 돌이 놓인 위치에 원을 그린다.

```cpp
void DrawStones(HDC hdc)
{
    constexpr int BOARD_SIZE = 19;
    constexpr int CELL_SIZE = 30;
    constexpr int MARGIN = 40;
    constexpr int STONE_RADIUS = 13;

    std::lock_guard<std::mutex> lock(g_gameState.mtx);

    for (int row = 0; row < BOARD_SIZE; ++row) {
        for (int col = 0; col < BOARD_SIZE; ++col) {
            int stone = g_gameState.board[row][col];
            if (stone == 0) continue;

            int x = MARGIN + col * CELL_SIZE;
            int y = MARGIN + row * CELL_SIZE;

            HBRUSH brush = CreateSolidBrush(
                stone == 1 ? RGB(0, 0, 0) : RGB(255, 255, 255));
            HBRUSH oldBrush = (HBRUSH)SelectObject(hdc, brush);

            HPEN pen = CreatePen(PS_SOLID, 1, RGB(0, 0, 0));
            HPEN oldPen = (HPEN)SelectObject(hdc, pen);

            Ellipse(hdc, x - STONE_RADIUS, y - STONE_RADIUS,
                         x + STONE_RADIUS, y + STONE_RADIUS);

            SelectObject(hdc, oldPen);
            DeleteObject(pen);
            SelectObject(hdc, oldBrush);
            DeleteObject(brush);
        }
    }
}
```

**상태 메시지 표시**

화면 하단에 현재 게임 상태를 텍스트로 표시한다. 서버 연결 여부, 대기 중, 내 턴, 상대 턴, 승리/패배 등의 메시지를 출력한다.

```cpp
void DrawStatus(HDC hdc, const RECT& clientRect)
{
    std::lock_guard<std::mutex> lock(g_gameState.mtx);

    SetBkMode(hdc, TRANSPARENT);
    SetTextColor(hdc, RGB(0, 0, 0));
    
    RECT textRect = clientRect;
    textRect.top = clientRect.bottom - 80;
    textRect.bottom = clientRect.bottom - 20;

    DrawText(hdc, g_gameState.statusMessage.c_str(), -1, &textRect,
             DT_CENTER | DT_VCENTER | DT_SINGLELINE);
}
```

이렇게 GDI를 사용하여 오목판, 돌, 상태 메시지를 화면에 렌더링한다. `InvalidateRect` 함수를 호출하면 `WM_PAINT` 메시지가 발생하여 화면이 다시 그려진다.

## 13.3 마우스 입력 처리
사용자는 마우스 클릭으로 오목판의 교차점을 선택하여 돌을 놓는다. `WM_LBUTTONDOWN` 메시지를 처리하여 마우스 좌표를 오목판의 행/열 인덱스로 변환하고, 서버에 돌 놓기 패킷을 전송한다.

**좌표 변환**

마우스 클릭 좌표(픽셀 단위)를 오목판의 행/열 인덱스(0~18)로 변환한다. 클릭 위치에서 가장 가까운 교차점을 찾는다.

```cpp
bool ScreenToBoard(int screenX, int screenY, int& row, int& col)
{
    constexpr int CELL_SIZE = 30;
    constexpr int MARGIN = 40;
    constexpr int BOARD_SIZE = 19;

    // 마진 제거
    int relX = screenX - MARGIN;
    int relY = screenY - MARGIN;

    // 가장 가까운 교차점 찾기
    col = (relX + CELL_SIZE / 2) / CELL_SIZE;
    row = (relY + CELL_SIZE / 2) / CELL_SIZE;

    // 범위 체크
    if (row < 0 || row >= BOARD_SIZE || col < 0 || col >= BOARD_SIZE) {
        return false;
    }

    return true;
}
```

**클릭 처리**

마우스 클릭 시, 현재 게임 상태를 확인하고 유효한 수인지 검사한 뒤 서버에 패킷을 전송한다.

```cpp
void OnLButtonDown(HWND hwnd, int x, int y)
{
    int row, col;
    if (!ScreenToBoard(x, y, row, col)) {
        return;  // 보드 범위 밖
    }

    std::lock_guard<std::mutex> lock(g_gameState.mtx);

    // 게임 시작 여부 확인
    if (!g_gameState.isGameStarted || g_gameState.isGameOver) {
        MessageBox(hwnd, L"게임이 진행 중이 아닙니다.", L"알림", MB_OK);
        return;
    }

    // 내 턴인지 확인
    if (g_gameState.currentTurn != g_gameState.myStone) {
        MessageBox(hwnd, L"상대방의 턴입니다.", L"알림", MB_OK);
        return;
    }

    // 이미 돌이 놓여 있는지 확인
    if (g_gameState.board[row][col] != 0) {
        MessageBox(hwnd, L"이미 돌이 놓여 있습니다.", L"알림", MB_OK);
        return;
    }

    // 서버에 돌 놓기 패킷 전송
    SendPlaceStonePacket(row, col);
}
```

서버에 패킷을 전송한 후, 서버의 응답을 기다린다. 서버가 유효한 수라고 판단하면 게임 상태 업데이트 패킷을 보내고, 클라이언트는 이를 받아 화면을 갱신한다.

## 13.4 네트워크 스레드 구현
네트워크 통신은 별도의 스레드에서 처리하여 UI가 블로킹되지 않도록 한다. 네트워크 스레드는 서버에 연결하고, 패킷을 송수신하며, 받은 패킷을 파싱하여 게임 상태를 업데이트한다.

**소켓 초기화 및 연결**

Winsock을 초기화하고 서버에 연결한다. 클라이언트는 간단한 블로킹 소켓을 사용한다.

```cpp
SOCKET g_socket = INVALID_SOCKET;

bool ConnectToServer(const char* serverIP, int port)
{
    WSADATA wsaData;
    if (WSAStartup(MAKEWORD(2, 2), &wsaData) != 0) {
        return false;
    }

    g_socket = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);
    if (g_socket == INVALID_SOCKET) {
        WSACleanup();
        return false;
    }

    sockaddr_in serverAddr = {};
    serverAddr.sin_family = AF_INET;
    serverAddr.sin_port = htons(port);
    inet_pton(AF_INET, serverIP, &serverAddr.sin_addr);

    if (connect(g_socket, (sockaddr*)&serverAddr, sizeof(serverAddr)) == SOCKET_ERROR) {
        closesocket(g_socket);
        WSACleanup();
        return false;
    }

    return true;
}
```

**네트워크 스레드 함수**

네트워크 스레드는 서버로부터 데이터를 수신하고, 패킷을 파싱하여 적절한 핸들러를 호출한다.

```cpp
void NetworkThreadFunc(HWND hwnd)
{
    if (!ConnectToServer("127.0.0.1", 9000)) {
        MessageBox(hwnd, L"서버 연결 실패", L"Error", MB_OK);
        return;
    }

    // 연결 성공 메시지 업데이트
    {
        std::lock_guard<std::mutex> lock(g_gameState.mtx);
        g_gameState.statusMessage = L"서버 연결 완료. 매칭 대기 중...";
    }
    InvalidateRect(hwnd, nullptr, FALSE);

    // 수신 버퍼
    std::vector<char> recvBuffer;
    recvBuffer.reserve(4096);

    char tempBuffer[4096];
    while (true) {
        int bytesReceived = recv(g_socket, tempBuffer, sizeof(tempBuffer), 0);
        if (bytesReceived <= 0) {
            // 연결 종료 또는 오류
            break;
        }

        recvBuffer.insert(recvBuffer.end(), tempBuffer, tempBuffer + bytesReceived);

        // 패킷 파싱
        while (recvBuffer.size() >= sizeof(PacketHeader)) {
            PacketHeader* header = (PacketHeader*)recvBuffer.data();
            if (recvBuffer.size() < header->size) {
                break;  // 아직 패킷이 완전히 도착하지 않음
            }

            // 패킷 처리
            ProcessPacket(hwnd, recvBuffer.data(), header->size);

            // 처리한 패킷 제거
            recvBuffer.erase(recvBuffer.begin(), recvBuffer.begin() + header->size);
        }
    }

    closesocket(g_socket);
    WSACleanup();
}
```

**패킷 송신**

서버에 패킷을 전송하는 헬퍼 함수를 구현한다. 송신 시에도 뮤텍스로 보호하여 여러 스레드에서 동시에 전송하지 않도록 한다.

```cpp
std::mutex g_sendMutex;

bool SendPacket(const void* data, int size)
{
    std::lock_guard<std::mutex> lock(g_sendMutex);

    int totalSent = 0;
    const char* ptr = (const char*)data;

    while (totalSent < size) {
        int sent = send(g_socket, ptr + totalSent, size - totalSent, 0);
        if (sent == SOCKET_ERROR) {
            return false;
        }
        totalSent += sent;
    }

    return true;
}

void SendPlaceStonePacket(int row, int col)
{
    PlaceStonePacket packet;
    packet.header.type = PacketType::PlaceStone;
    packet.header.size = sizeof(PlaceStonePacket);
    packet.row = row;
    packet.col = col;
```

## 13.5 게임 화면 업데이트
네트워크 스레드에서 서버로부터 패킷을 받아 게임 상태를 업데이트하면, UI 스레드에 화면을 다시 그리도록 요청해야 한다. Win32에서는 `InvalidateRect` 함수를 호출하여 `WM_PAINT` 메시지를 발생시킨다.

**게임 시작 패킷 처리**

서버가 매칭을 완료하고 게임 시작 패킷을 보내면, 내가 조종할 돌의 색상(흑/백)과 상대방 정보를 받는다.

```cpp
    g_gameState.currentTurn = 1;  // 흑돌이 먼저 시작

    if (g_gameState.myStone == 1) {
        g_gameState.statusMessage = L"게임 시작! 당신은 흑돌입니다. 당신의 턴입니다.";
    } else {
        g_gameState.statusMessage = L"게임 시작! 당신은 백돌입니다. 상대방의 턴입니다.";
    }

    InvalidateRect(hwnd, nullptr, FALSE);
}
```

**돌 놓기 결과 패킷 처리**

서버는 누군가 돌을 놓으면 모든 클라이언트에게 돌 위치와 다음 턴 정보를 브로드캐스트한다.


**게임 종료 패킷 처리**

게임이 끝나면 서버는 승자 정보를 보낸다. 클라이언트는 승리/패배/무승부 메시지를 표시한다.

```cpp
void OnGameEnd(HWND hwnd, GameEndPacket* packet)
{
    std::lock_guard<std::mutex> lock(g_gameState.mtx);

    g_gameState.isGameOver = true;

    if (packet->winner == g_gameState.myStone) {
        g_gameState.statusMessage = L"승리했습니다!";
    } else if (packet->winner == 0) {
        g_gameState.statusMessage = L"무승부입니다.";
    } else {
        g_gameState.statusMessage = L"패배했습니다.";
    }

    InvalidateRect(hwnd, nullptr, FALSE);

    MessageBox(hwnd, g_gameState.statusMessage.c_str(), L"게임 종료", MB_OK);
}
```

`InvalidateRect` 함수는 윈도우의 클라이언트 영역을 무효화하여 다시 그리도록 요청한다. 두 번째 매개변수로 `nullptr`를 전달하면 전체 영역이 무효화되고, 세 번째 매개변수 `FALSE`는 배경을 지우지 않는다는 의미다.

```cpp
bool ScreenToBoard(int screenX, int screenY, int& row, int& col)
{
    int relX = screenX - MARGIN;
    int relY = screenY - MARGIN;

    col = (relX + CELL_SIZE / 2) / CELL_SIZE;
    row = (relY + CELL_SIZE / 2) / CELL_SIZE;

    if (row < 0 || row >= BOARD_SIZE || col < 0 || col >= BOARD_SIZE) {

    }

    return true;
}

bool ConnectToServer(const char* serverIP, int port)
{
    WSADATA wsaData;
    if (WSAStartup(MAKEWORD(2, 2), &wsaData) != 0) {
        return false;
    }

    g_socket = socket(AF_INET, SOCK_STREAM, IPPROTO_TCP);
    if (g_socket == INVALID_SOCKET) {
        WSACleanup();
        return false;
    }

    sockaddr_in serverAddr = {};
    serverAddr.sin_family = AF_INET;
    serverAddr.sin_port = htons(static_cast<u_short>(port));
    inet_pton(AF_INET, serverIP, &serverAddr.sin_addr);

    if (connect(g_socket, (sockaddr*)&serverAddr, sizeof(serverAddr)) == SOCKET_ERROR) {
        closesocket(g_socket);
        g_socket = INVALID_SOCKET;
        WSACleanup();
        return false;
    }

    return true;
}

void NetworkThreadFunc(HWND hwnd)
{
    if (!ConnectToServer("127.0.0.1", 9000)) {
        MessageBox(hwnd, L"서버 연결 실패", L"Error", MB_OK);
        return;
    }

    {
        std::lock_guard<std::mutex> lock(g_gameState.mtx);
        g_gameState.statusMessage = L"서버 연결 완료. 매칭 대기 중...";
    }
    InvalidateRect(hwnd, nullptr, FALSE);

    std::vector<char> recvBuffer;
    recvBuffer.reserve(4096);

    char tempBuffer[4096];
    while (true) {
        int bytesReceived = recv(g_socket, tempBuffer, sizeof(tempBuffer), 0);
        if (bytesReceived <= 0) {
            break;
        }

        recvBuffer.insert(recvBuffer.end(), tempBuffer, tempBuffer + bytesReceived);

        while (recvBuffer.size() >= sizeof(PacketHeader)) {
            PacketHeader* header = (PacketHeader*)recvBuffer.data();
            if (recvBuffer.size() < header->size) {
                break;
            }

            ProcessPacket(hwnd, recvBuffer.data(), header->size);

            recvBuffer.erase(recvBuffer.begin(), recvBuffer.begin() + header->size);
        }
    }

    {
        std::lock_guard<std::mutex> lock(g_gameState.mtx);
        g_gameState.statusMessage = L"서버 연결 종료";
    }
    InvalidateRect(hwnd, nullptr, FALSE);

    closesocket(g_socket);
    g_socket = INVALID_SOCKET;
}

void ProcessPacket(HWND hwnd, const char* data, int size)
{

    switch (header->type) {
    case PacketType::GameStart:

        break;
    case PacketType::PlaceStoneResult:
        OnPlaceStoneResult(hwnd, (PlaceStoneResultPacket*)data);
        break;
    case PacketType::GameEnd:
        OnGameEnd(hwnd, (GameEndPacket*)data);
        break;
    }
}

bool SendPacket(const void* data, int size)
{
    std::lock_guard<std::mutex> lock(g_sendMutex);

    if (g_socket == INVALID_SOCKET) {
        return false;
    }

    int totalSent = 0;
    const char* ptr = (const char*)data;

    while (totalSent < size) {
        int sent = send(g_socket, ptr + totalSent, size - totalSent, 0);
        if (sent == SOCKET_ERROR) {
            return false;
        }
        totalSent += sent;
    }

    return true;
}

void SendPlaceStonePacket(int row, int col)
{
    PlaceStonePacket packet;
    packet.header.type = PacketType::PlaceStone;
    packet.header.size = sizeof(PlaceStonePacket);
    packet.row = row;
    packet.col = col;

    SendPacket(&packet, sizeof(packet));
}
```

**빌드 및 실행**

Visual Studio 2022에서 Win32 프로젝트를 생성하고, 위 코드를 추가한다. 프로젝트 속성에서 C++ 언어 표준을 C++20 이상으로 설정하고, Winsock 라이브러리(`ws2_32.lib`)를 링커에 추가한다. 서버를 먼저 실행한 후, 클라이언트를 두 개 실행하면 자동으로 매칭되어 게임이 시작된다.

클라이언트는 서버의 `127.0.0.1:9000` 주소로 연결을 시도한다. 필요에 따라 IP 주소와 포트를 수정할 수 있다. 실제 배포 시에는 설정 파일이나 UI에서 서버 주소를 입력받도록 개선할 수 있다.

이 장에서는 Win32 API를 사용하여 오목 게임 클라이언트를 구현했다. GDI로 오목판과 돌을 렌더링하고, 마우스 입력을 처리하며, 별도의 네트워크 스레드에서 서버와 통신하여 실시간 게임 플레이를 구현했다. 클라이언트는 최대한 간결하게 작성했지만, 실제 게임을 테스트하고 플레이하기에 충분한 기능을 갖추고 있다.
