효율적인 버퍼 관리는 게임 서버 성능의 핵심이다. 버퍼를 잘못 설계하면 불필요한 메모리 복사가 빈번하게 발생하고, 메모리 단편화가 심해지며, 결국 서버의 처리량이 크게 저하된다. 이번 챕터에서는 링 버퍼(Ring Buffer) 구조를 구현하여 이러한 문제를 해결하고, 송수신 버퍼를 효율적으로 관리하는 방법을 학습한다.

## 4.1 링 버퍼(Ring Buffer) 구조 이해

**선형 버퍼의 문제점**  
가장 단순한 버퍼 구현은 고정 크기의 배열을 사용하는 선형 버퍼다. 데이터를 배열의 앞쪽부터 채워나가고, 데이터를 소비하면 뒤쪽 데이터를 앞으로 이동시키는 방식이다. 하지만 이 방식은 치명적인 성능 문제가 있다.

```cpp
// 선형 버퍼의 비효율적인 예
class LinearBuffer {
    char buffer_[4096];
    int dataSize_;
    
    void Consume(int size) {
        dataSize_ -= size;
        // 남은 데이터를 앞으로 이동 (비효율적!)
        memmove(buffer_, buffer_ + size, dataSize_);
    }
};
```

위 코드의 `Consume` 함수는 데이터를 소비할 때마다 `memmove`를 호출하여 나머지 데이터를 배열의 앞쪽으로 이동시킨다. 만약 4KB 버퍼에 3KB의 데이터가 있고, 100바이트만 소비한다면 2.9KB의 데이터를 복사해야 한다. 이러한 복사는 CPU 사이클을 낭비하고, 캐시 효율성도 떨어뜨린다.  
또 다른 문제는 버퍼가 꽉 찼을 때다. 선형 버퍼에서는 일부 데이터를 소비해도 버퍼의 뒤쪽에만 공간이 생기므로, 연속된 공간이 필요한 작업에는 사용할 수 없다. 결국 동적 메모리 할당이나 추가적인 복사가 필요해진다.

**링 버퍼의 원리**  
링 버퍼는 배열을 원형으로 간주하여 읽기 위치(Read Position)와 쓰기 위치(Write Position)를 추적하는 방식이다. 쓰기 위치가 배열의 끝에 도달하면 다시 처음으로 돌아가고, 읽기 위치도 마찬가지다. 이를 통해 데이터 이동 없이 효율적으로 버퍼를 관리할 수 있다.

```
[링 버퍼의 구조]

         Write
           ↓
    ┌──┬──┬──┬──┬──┬──┬──┬──┐
    │  │  │DD│DD│DD│DD│  │  │
    └──┴──┴──┴──┴──┴──┴──┴──┘
              ↑
            Read

데이터 영역: Read부터 Write 직전까지
빈 공간: Write부터 Read 직전까지

[쓰기 위치가 배열 끝을 넘어갈 때]

    Read        Write
     ↓            ↓
    ┌──┬──┬──┬──┬──┬──┬──┬──┐
    │DD│DD│  │  │  │  │DD│DD│
    └──┴──┴──┴──┴──┴──┴──┴──┘
    
데이터가 두 영역으로 분리됨
```

링 버퍼는 읽기/쓰기 포인터만 이동하므로 데이터 복사가 필요 없다. 버퍼가 순환하면서 재사용되므로 메모리 효율성이 높고, 빈 공간을 최대한 활용할 수 있다. 다만 데이터가 배열의 끝과 시작 부분에 걸쳐 있을 수 있어서, 연속된 메모리 공간이 필요한 경우 주의가 필요하다.

**링 버퍼의 주요 연산**

링 버퍼에서 지원해야 할 주요 연산은 다음과 같다. 첫째, 사용 가능한 빈 공간 크기 계산이다. 둘째, 쓰기 가능한 연속된 공간의 포인터와 크기 반환이다. 셋째, 데이터 쓰기 후 쓰기 포인터 이동이다. 넷째, 읽을 수 있는 데이터 크기 계산이다. 다섯째, 읽기 가능한 연속된 데이터의 포인터와 크기 반환이다. 여섯째, 데이터 읽기 후 읽기 포인터 이동이다.

```cpp
class RingBuffer {
private:
    std::vector<BYTE> buffer_;
    int capacity_;
    int readPos_;
    int writePos_;
    
public:
    // 사용 가능한 빈 공간 크기
    int GetFreeSize() const {
        if (writePos_ >= readPos_) {
            return capacity_ - (writePos_ - readPos_) - 1;
        } else {
            return readPos_ - writePos_ - 1;
        }
    }
    
    // 사용 중인 데이터 크기
    int GetDataSize() const {
        if (writePos_ >= readPos_) {
            return writePos_ - readPos_;
        } else {
            return capacity_ - (readPos_ - writePos_);
        }
    }
};
```

링 버퍼를 설계할 때 중요한 점은 읽기 위치와 쓰기 위치가 같을 때 비어있는지 꽉 찬 것인지 구분하는 것이다. 일반적으로 두 가지 방법을 사용한다. 첫 번째는 버퍼의 한 칸을 항상 비워두는 방법이다. 이 경우 `(writePos + 1) % capacity == readPos`이면 버퍼가 가득 찬 것이다. 두 번째는 별도의 플래그나 카운터를 사용하는 방법이다. 여기서는 첫 번째 방법을 사용한다.

## 4.2 송신 버퍼 구현 (Send Buffer)

**송신 버퍼의 역할**  
송신 버퍼는 애플리케이션이 보내고자 하는 데이터를 임시로 저장하는 공간이다. IOCP 환경에서는 비동기 송신(`WSASend`)이 완료될 때까지 데이터를 보존해야 한다. 또한 여러 개의 작은 패킷을 모아서 한 번에 보내는 버퍼링 기능도 중요하다.

송신 버퍼의 주요 책임은 다음과 같다. 첫째, 애플리케이션 레이어에서 보내려는 데이터를 버퍼에 추가한다. 둘째, 버퍼에 저장된 데이터로 `WSASend`를 호출한다. 셋째, `WSASend` 완료 후 전송된 데이터만큼 버퍼에서 제거한다. 넷째, 여러 개의 작은 패킷을 모아서 한 번에 전송하여 시스템 콜 횟수를 줄인다.

```
[송신 버퍼 동작 흐름]

┌─────────────┐
│ Application │
└──────┬──────┘
       │ Send(data)
       ↓
┌─────────────────┐
│  Send Buffer    │
│  [Data Queue]   │
└────────┬────────┘
         │ WSASend()
         ↓
┌─────────────────┐
│  TCP Send      │
│  Buffer        │
└────────┬────────┘
         │
         ↓
    [Network]
```

**링 버퍼 기반 송신 버퍼**   
송신 버퍼를 링 버퍼로 구현하면 효율적인 메모리 관리가 가능하다. 데이터를 추가할 때는 쓰기 포인터 위치에 복사하고, 전송이 완료되면 읽기 포인터를 이동시키기만 하면 된다.

```cpp
class SendBuffer {
private:
    std::vector<BYTE> buffer_;
    int capacity_;
    int readPos_;
    int writePos_;
    std::mutex mutex_;
    bool sendInProgress_;
    
public:
    SendBuffer(int capacity)
        : buffer_(capacity)
        , capacity_(capacity)
        , readPos_(0)
        , writePos_(0)
        , sendInProgress_(false)
    {}
    
    // 데이터 추가
    bool Write(const BYTE* data, int size) {
        std::lock_guard<std::mutex> lock(mutex_);
        
        if (GetFreeSize() < size) {
            return false;  // 버퍼 부족
        }
        
        int firstSize = std::min(size, capacity_ - writePos_);
        memcpy(&buffer_[writePos_], data, firstSize);
        
        if (size > firstSize) {
            // 배열 끝을 넘어가는 경우
            memcpy(&buffer_[0], data + firstSize, size - firstSize);
        }
        
        writePos_ = (writePos_ + size) % capacity_;
        return true;
    }
    
    // 송신 데이터 준비
    void PrepareData(WSABUF wsaBufs[2], int& bufferCount) {
        std::lock_guard<std::mutex> lock(mutex_);
        
        int dataSize = GetDataSize();
        if (dataSize == 0) {
            bufferCount = 0;
            return;
        }
        
        // 첫 번째 버퍼
        int firstSize = std::min(dataSize, capacity_ - readPos_);
        wsaBufs[0].buf = (char*)&buffer_[readPos_];
        wsaBufs[0].len = firstSize;
        bufferCount = 1;
        
        // 두 번째 버퍼 (데이터가 배열 끝을 넘어가는 경우)
        if (dataSize > firstSize) {
            wsaBufs[1].buf = (char*)&buffer_[0];
            wsaBufs[1].len = dataSize - firstSize;
            bufferCount = 2;
        }
        
        sendInProgress_ = true;
    }
    
    // 송신 완료 처리
    void OnSendCompleted(int bytesSent) {
        std::lock_guard<std::mutex> lock(mutex_);
        readPos_ = (readPos_ + bytesSent) % capacity_;
        sendInProgress_ = false;
    }
    
private:
    int GetFreeSize() const {
        if (writePos_ >= readPos_) {
            return capacity_ - (writePos_ - readPos_) - 1;
        } else {
            return readPos_ - writePos_ - 1;
        }
    }
    
    int GetDataSize() const {
        if (writePos_ >= readPos_) {
            return writePos_ - readPos_;
        } else {
            return capacity_ - (readPos_ - writePos_);
        }
    }
};
```

**송신 직렬화 보장**  
IOCP 환경에서는 여러 워커 스레드가 동시에 세션에 접근할 수 있다. 하나의 세션에 대해 동시에 여러 개의 `WSASend`를 호출하면 데이터 순서가 보장되지 않는다. 따라서 한 번에 하나의 송신만 진행되도록 직렬화해야 한다.

```cpp
class Session {
private:
    SendBuffer sendBuffer_;
    std::atomic<bool> sendRegistered_;
    
public:
    void Send(const BYTE* data, int size) {
        // 송신 버퍼에 데이터 추가
        if (!sendBuffer_.Write(data, size)) {
            std::cout << "송신 버퍼 부족" << std::endl;
            Disconnect();
            return;
        }
        
        // 이미 송신 중이면 등록만 하고 리턴
        if (sendRegistered_.exchange(true) == true) {
            return;
        }
        
        // 실제 송신 시작
        RegisterSend();
    }
    
private:
    void RegisterSend() {
        WSABUF wsaBufs[2];
        int bufferCount = 0;
        
        sendBuffer_.PrepareData(wsaBufs, bufferCount);
        
        if (bufferCount == 0) {
            sendRegistered_ = false;
            return;
        }
        
        SendOverlapped* sendOv = new SendOverlapped;
        ZeroMemory(&sendOv->overlapped, sizeof(OVERLAPPED));
        sendOv->owner = this;
        
        AddRef();
        
        DWORD bytesSent = 0;
        int result = WSASend(socket_, wsaBufs, bufferCount,
                            &bytesSent, 0,
                            &sendOv->overlapped, nullptr);
        
        if (result == SOCKET_ERROR) {
            int error = WSAGetLastError();
            if (error != WSA_IO_PENDING) {
                Release();
                delete sendOv;
                sendRegistered_ = false;
                Disconnect();
            }
        }
    }
    
    void ProcessSend(SendOverlapped* sendOv, DWORD bytesTransferred) {
        delete sendOv;
        
        // 송신 버퍼에서 전송된 데이터 제거
        sendBuffer_.OnSendCompleted(bytesTransferred);
        
        // 추가로 보낼 데이터가 있는지 확인
        RegisterSend();
    }
};
```

위 코드에서 `sendRegistered_` 플래그를 사용하여 송신 작업을 직렬화한다. 여러 스레드가 동시에 `Send`를 호출해도 `exchange` 연산을 통해 하나의 스레드만 실제 송신을 시작한다. 나머지 스레드는 데이터만 버퍼에 추가하고 리턴한다. 송신이 완료되면 `ProcessSend`에서 `RegisterSend`를 다시 호출하여 버퍼에 남아있는 데이터를 계속 전송한다.

### IOCP 환경에서 송신(Send) 직렬화가 필요한 이유

**IOCP의 기본 동작 구조 이해**  
IOCP(I/O Completion Port)는 Windows의 고성능 비동기 I/O 모델로, 여러 개의 워커 스레드가 하나의 Completion Port 큐를 공유하며 완료된 I/O 이벤트를 처리한다. 이 구조에서는 **어떤 워커 스레드든 어떤 세션의 이벤트든 처리할 수 있기 때문에**, 단일 세션에 대해서도 여러 스레드가 동시에 접근하는 상황이 자연스럽게 발생한다.

예를 들어 Session A에 대한 수신 완료 이벤트를 스레드 1이 처리하는 동안, 다른 곳에서 Session A로 데이터를 보내야 하는 상황이 생기면 스레드 2, 3이 동시에 `WSASend`를 호출할 수 있다.  
    
    
**핵심 문제: WSASend를 여러 번 동시에 호출하면 무슨 일이 생기는가**  
`WSASend`는 비동기 함수이다. 호출 시점에 데이터를 즉시 전송하는 것이 아니라, **OS의 내부 송신 큐에 전송 요청을 등록**하고 반환한다. 문제는 여기서 시작된다.

**OS 레벨의 전송 순서 비보장** 문제가 가장 근본적인 원인이다. Microsoft 공식 문서에서도 명시하고 있듯이, 하나의 소켓에 대해 동시에 복수의 `WSASend`가 걸려 있을 경우 **전송 완료 순서는 보장되지 않는다.** OS가 내부적으로 어떤 요청을 먼저 처리할지는 스케줄링 상황에 따라 달라지기 때문이다.

실제 시나리오를 예로 들면, 스레드 1이 `[패킷 A]`를, 스레드 2가 `[패킷 B]`를 거의 동시에 `WSASend`로 요청했을 때, 수신 측에는 `[패킷 B] → [패킷 A]` 순서로 도달할 수 있다. 게임 서버처럼 패킷 순서가 로직에 직접적인 영향을 미치는 환경에서는 이것이 **치명적인 버그**로 이어진다.  

  
**더 깊은 문제: WSABUF와 메모리 생명주기**  
`WSASend`는 비동기이므로, 호출 후 즉시 반환되더라도 **OS가 해당 버퍼를 실제로 다 읽을 때까지 버퍼는 유효하게 유지**되어야 한다. 여러 스레드가 동시에 각자의 버퍼로 `WSASend`를 호출하면, 각 버퍼의 생명주기 관리가 복잡해지고 **Use-After-Free** 버그나 버퍼 해제 타이밍 문제가 발생할 수 있다.  
  

**또 다른 문제: 송신 버퍼(Send Ring Buffer)의 동시 접근**  
일반적으로 세션은 내부적으로 **링 버퍼(Ring Buffer)** 혹은 **큐(Queue)** 형태의 송신 버퍼를 보유한다. 여러 스레드가 동시에 이 버퍼에 데이터를 쓰려고 하면, 적절한 동기화 없이는 데이터가 뒤섞이거나 버퍼 포인터(Head/Tail)가 오염되는 **Race Condition**이 발생힌다. 이는 데이터 손상 혹은 크래시로 직결된다.  
   
  
**올바른 해결 전략: 송신 직렬화 패턴**  
이 문제를 해결하기 위해 실무에서 가장 널리 쓰이는 패턴은 **"Post → Flag Check → Send"** 패턴이다. 동작 원리는 다음과 같다.  

```cpp
void Session::Send(BufferPtr data) {
    // 1. 락을 잡고 송신 큐에 데이터를 안전하게 삽입
    {
        std::lock_guard<std::mutex> lock(sendLock_);
        sendQueue_.push(data);
    }

    // 2. 현재 송신 중인지 플래그로 확인 (atomic CAS 사용)
    bool expected = false;
    if (isSending_.compare_exchange_strong(expected, true)) {
        // 3. 송신 중이 아닐 때만 실제 WSASend 호출
        RegisterSend();
    }
    // 송신 중이었다면 큐에만 넣고 반환 → 현재 송신 완료 후 자동으로 처리됨
}

void Session::OnSendComplete() {
    // 4. 송신 완료 콜백에서 큐에 남은 데이터 확인
    std::lock_guard<std::mutex> lock(sendLock_);
    if (sendQueue_.empty()) {
        isSending_.store(false); // 더 보낼 것이 없으면 플래그 해제
    } else {
        RegisterSend(); // 남은 데이터 이어서 전송
    }
}
```

이 패턴의 핵심은 **"WSASend는 항상 하나만 걸려 있다"** 는 불변 조건(Invariant)을 강제하는 것이다. 새로운 데이터는 항상 큐에 먼저 적재되고, 실제 전송은 이전 전송이 완료된 이후에만 트리거된다.
  
  
**정리: 직렬화가 필요한 이유 요약**  

| 문제 | 발생 원인 | 결과 |
|---|---|---|
| 패킷 순서 역전 | OS가 다중 WSASend 순서를 보장하지 않음 | 로직 버그, 프로토콜 파괴 |
| 버퍼 생명주기 오염 | 비동기 전송 중 버퍼가 해제됨 | Use-After-Free, 크래시 |
| 송신 버퍼 Race Condition | 다중 스레드의 동시 버퍼 접근 | 데이터 오염, 크래시 |

결국 **"IOCP는 병렬 처리를 위한 구조이지만, 단일 세션의 송신만큼은 반드시 직렬화되어야 한다"** 는 원칙이 성립한다. 이것은 IOCP의 한계가 아니라, TCP 소켓의 스트림 특성과 OS 비동기 I/O 모델이 결합되어 발생하는 필연적인 설계 요구사항이다.

## 4.3 수신 버퍼 구현 (Recv Buffer)

**수신 버퍼의 역할**  
수신 버퍼는 소켓으로부터 받은 데이터를 임시로 저장하고, 완전한 패킷을 조립하는 역할을 한다. TCP는 스트림 기반이므로, 송신 측에서 100바이트를 보냈더라도 수신 측에서는 여러 번에 걸쳐 받을 수 있다. 수신 버퍼는 이러한 부분 수신(Partial Receive)을 처리하여 완전한 패킷을 만들어낸다.

수신 버퍼의 주요 책임은 다음과 같다. 첫째, `WSARecv`로 받은 데이터를 버퍼에 추가한다. 둘째, 버퍼에서 완전한 패킷을 추출한다. 셋째, 패킷 경계를 정확히 구분한다. 넷째, 패킷 처리 후 버퍼에서 해당 데이터를 제거한다.

```
[수신 버퍼 동작 흐름]

    [Network]
         ↓
┌─────────────────┐
│  TCP Recv      │
│  Buffer        │
└────────┬────────┘
         │ WSARecv()
         ↓
┌─────────────────┐
│  Recv Buffer    │  → Partial data 1
│  [Ring Buffer]  │  → Partial data 2
└────────┬────────┘  → Complete Packet!
         │
         ↓
┌─────────────────┐
│ Packet Handler  │
└─────────────────┘
```

**링 버퍼 기반 수신 버퍼**  
수신 버퍼도 링 버퍼로 구현한다. `WSARecv`는 링 버퍼의 쓰기 가능한 공간에 직접 데이터를 받고, 패킷 파서는 읽기 위치부터 데이터를 읽어 패킷을 조립한다.

```cpp
class RecvBuffer {
private:
    std::vector<BYTE> buffer_;
    int capacity_;
    int readPos_;
    int writePos_;
    
public:
    RecvBuffer(int capacity)
        : buffer_(capacity)
        , capacity_(capacity)
        , readPos_(0)
        , writePos_(0)
    {}
    
    // WSARecv를 위한 버퍼 준비
    void GetWriteBuffer(WSABUF& wsaBuf) {
        int freeSize = GetFreeSize();
        
        if (writePos_ >= readPos_) {
            // 쓰기 위치가 읽기 위치보다 뒤에 있음
            // 배열 끝까지만 사용 (단순화)
            wsaBuf.buf = (char*)&buffer_[writePos_];
            wsaBuf.len = capacity_ - writePos_;
            
            // 읽기 위치가 0이면 한 칸 남겨야 함
            if (readPos_ == 0) {
                wsaBuf.len -= 1;
            }
        } else {
            // 쓰기 위치가 읽기 위치보다 앞에 있음
            wsaBuf.buf = (char*)&buffer_[writePos_];
            wsaBuf.len = readPos_ - writePos_ - 1;
        }
    }
    
    // 수신 완료 처리
    bool OnRecvCompleted(int bytesReceived) {
        if (GetFreeSize() < bytesReceived) {
            return false;  // 버퍼 오버플로우
        }
        
        writePos_ = (writePos_ + bytesReceived) % capacity_;
        return true;
    }
    
    // 데이터 읽기 (Peek)
    bool Peek(BYTE* dest, int size) {
        if (GetDataSize() < size) {
            return false;  // 데이터 부족
        }
        
        int firstSize = std::min(size, capacity_ - readPos_);
        memcpy(dest, &buffer_[readPos_], firstSize);
        
        if (size > firstSize) {
            memcpy(dest + firstSize, &buffer_[0], size - firstSize);
        }
        
        return true;
    }
    
    // 데이터 소비
    void Consume(int size) {
        readPos_ = (readPos_ + size) % capacity_;
    }
    
    // 연속된 데이터 포인터 (패킷 헤더 읽기용)
    BYTE* GetDataPtr() {
        return &buffer_[readPos_];
    }
    
    int GetContinuousDataSize() const {
        if (writePos_ >= readPos_) {
            return writePos_ - readPos_;
        } else {
            return capacity_ - readPos_;
        }
    }
    
    int GetDataSize() const {
        if (writePos_ >= readPos_) {
            return writePos_ - readPos_;
        } else {
            return capacity_ - (readPos_ - writePos_);
        }
    }
    
    int GetFreeSize() const {
        if (writePos_ >= readPos_) {
            return capacity_ - (writePos_ - readPos_) - 1;
        } else {
            return readPos_ - writePos_ - 1;
        }
    }
};
```

**패킷 경계 구분**  
TCP는 패킷의 경계를 보장하지 않으므로, 애플리케이션 레벨에서 패킷의 시작과 끝을 구분해야 한다. 가장 일반적인 방법은 고정 길이 헤더를 사용하는 것이다. 헤더에 패킷의 전체 크기를 포함시켜, 수신 버퍼에 충분한 데이터가 쌓일 때까지 기다린다.

```cpp
// 패킷 헤더 (간단한 예)
#pragma pack(push, 1)
struct PacketHeader {
    uint16_t size;      // 헤더 포함 전체 크기
    uint16_t packetId;  // 패킷 타입
};
#pragma pack(pop)

class Session {
private:
    RecvBuffer recvBuffer_;
    
    void ProcessRecv(DWORD bytesTransferred) {
        if (!recvBuffer_.OnRecvCompleted(bytesTransferred)) {
            std::cout << "수신 버퍼 오버플로우" << std::endl;
            Disconnect();
            return;
        }
        
        // 패킷 파싱
        while (true) {
            // 헤더를 읽을 수 있는지 확인
            if (recvBuffer_.GetDataSize() < sizeof(PacketHeader)) {
                break;
            }
            
            // 헤더 읽기 (Peek)
            PacketHeader header;
            if (recvBuffer_.GetContinuousDataSize() >= sizeof(PacketHeader)) {
                // 연속된 공간에 있으면 직접 포인터 사용
                header = *(PacketHeader*)recvBuffer_.GetDataPtr();
            } else {
                // 배열 경계를 넘어가면 복사
                recvBuffer_.Peek((BYTE*)&header, sizeof(PacketHeader));
            }
            
            // 패킷 전체를 받았는지 확인
            if (recvBuffer_.GetDataSize() < header.size) {
                break;
            }
            
            // 패킷 처리
            HandlePacket(header);
            
            // 버퍼에서 패킷 제거
            recvBuffer_.Consume(header.size);
        }
        
        // 다음 수신 대기
        PostRecv();
    }
    
    void HandlePacket(const PacketHeader& header) {
        // 패킷 타입에 따라 처리
        // Chapter 5, 6에서 자세히 다룸
    }
};
```
  
  
## 4.4 Partial Send/Recv 처리

**Partial Send 문제**  
`WSASend`는 요청한 데이터를 모두 전송하지 못할 수 있다. 특히 송신 버퍼가 가득 찬 경우, 일부만 전송하고 나머지는 나중에 전송해야 한다. `WSASend` 완료 통지의 `bytesTransferred` 값을 확인하여 실제로 전송된 바이트 수를 파악해야 한다.

```cpp
void Session::ProcessSend(SendOverlapped* sendOv, DWORD bytesTransferred) {
    delete sendOv;
    
    // 전송된 만큼 버퍼에서 제거
    sendBuffer_.OnSendCompleted(bytesTransferred);
    
    // 송신 버퍼에 데이터가 남아있으면 계속 전송
    if (sendBuffer_.GetDataSize() > 0) {
        RegisterSend();
    } else {
        sendRegistered_ = false;
    }
}
```

위 코드에서 `OnSendCompleted`는 실제로 전송된 바이트만큼만 읽기 포인터를 이동시킨다. 만약 요청한 데이터가 모두 전송되지 않았다면, 송신 버퍼에 데이터가 남아있으므로 `RegisterSend`를 다시 호출하여 나머지를 전송한다.

**Partial Recv 처리**  
수신의 경우는 더 복잡하다. `WSARecv`는 소켓 버퍼에 있는 데이터만큼만 반환하므로, 완전한 패킷을 받기 위해 여러 번의 수신이 필요할 수 있다. 수신 버퍼는 부분적으로 받은 데이터를 계속 축적하고, 완전한 패킷이 완성될 때까지 기다려야 한다.

```cpp
void Session::ProcessRecv(DWORD bytesTransferred) {
    UpdateRecvTime();
    
    if (bytesTransferred == 0) {
        Disconnect();
        return;
    }
    
    // 수신 버퍼에 데이터 추가
    if (!recvBuffer_.OnRecvCompleted(bytesTransferred)) {
        std::cout << "수신 버퍼 오버플로우" << std::endl;
        Disconnect();
        return;
    }
    
    // 패킷 파싱 (완전한 패킷만 처리)
    int processedBytes = 0;
    while (true) {
        // 헤더 크기 확인
        if (recvBuffer_.GetDataSize() < sizeof(PacketHeader)) {
            break;  // 헤더도 완성되지 않음
        }
        
        // 헤더 읽기
        PacketHeader header;
        if (recvBuffer_.GetContinuousDataSize() >= sizeof(PacketHeader)) {
            header = *(PacketHeader*)recvBuffer_.GetDataPtr();
        } else {
            recvBuffer_.Peek((BYTE*)&header, sizeof(PacketHeader));
        }
        
        // 패킷 크기 검증
        if (header.size < sizeof(PacketHeader) || 
            header.size > MAX_PACKET_SIZE) {
            std::cout << "잘못된 패킷 크기: " << header.size << std::endl;
            Disconnect();
            return;
        }
        
        // 패킷 전체가 도착했는지 확인
        if (recvBuffer_.GetDataSize() < header.size) {
            break;  // 아직 패킷이 완성되지 않음
        }
        
        // 패킷 처리
        OnPacket(header);
        
        // 버퍼에서 패킷 제거
        recvBuffer_.Consume(header.size);
        processedBytes += header.size;
    }
    
    // 다음 수신 대기
    PostRecv();
}
```

**버퍼 확장 전략**  
고정 크기 링 버퍼는 간단하고 효율적이지만, 큰 패킷을 처리할 때 문제가 될 수 있다. 버퍼 크기보다 큰 패킷이 오면 처리할 수 없다. 이런 경우 동적으로 버퍼를 확장하는 전략이 필요하다.

```cpp
class DynamicRecvBuffer {
private:
    std::vector<BYTE> buffer_;
    int readPos_;
    int writePos_;
    
    void Expand(int minSize) {
        int newCapacity = buffer_.size() * 2;
        while (newCapacity < minSize) {
            newCapacity *= 2;
        }
        
        std::vector<BYTE> newBuffer(newCapacity);
        
        // 기존 데이터 복사
        int dataSize = GetDataSize();
        if (writePos_ >= readPos_) {
            memcpy(&newBuffer[0], &buffer_[readPos_], dataSize);
        } else {
            int firstSize = buffer_.size() - readPos_;
            memcpy(&newBuffer[0], &buffer_[readPos_], firstSize);
            memcpy(&newBuffer[firstSize], &buffer_[0], writePos_);
        }
        
        buffer_ = std::move(newBuffer);
        readPos_ = 0;
        writePos_ = dataSize;
    }
    
public:
    void EnsureSpace(int size) {
        if (GetFreeSize() < size) {
            Expand(GetDataSize() + size);
        }
    }
};
```

하지만 동적 확장은 메모리 할당과 데이터 복사 비용이 발생하므로, 가능하면 초기 버퍼 크기를 충분히 크게 설정하는 것이 좋다. 일반적인 게임 서버에서는 64KB 정도의 고정 크기 버퍼로도 충분하다.

## 4.5 버퍼 크기 최적화 전략

**최적 버퍼 크기 결정**  
버퍼 크기는 성능과 메모리 사용량의 트레이드오프다. 버퍼가 너무 작으면 빈번한 송수신 작업이 필요하고, 버퍼가 너무 크면 메모리 낭비가 심하다. 최적의 크기는 게임의 특성과 트래픽 패턴에 따라 다르지만, 몇 가지 가이드라인이 있다.

수신 버퍼의 경우, 일반적으로 64KB(65536바이트)가 적절하다. 이는 TCP의 기본 윈도우 크기와 비슷하며, 대부분의 게임 패킷을 처리하기에 충분하다. MMO 게임처럼 큰 데이터를 전송하는 경우 128KB나 256KB로 늘릴 수 있다.

송신 버퍼의 경우, 게임의 송신 패턴에 따라 결정된다. 실시간 액션 게임은 작고 빈번한 패킷을 보내므로 32KB 정도면 충분하다. 반면 파일 전송이나 큰 데이터를 보내는 게임은 더 큰 버퍼가 필요하다.

```cpp

    static constexpr int FAST_ACTION_GAME_RECV_BUFFER = 32 * 1024;   // 32KB
    static constexpr int FAST_ACTION_GAME_SEND_BUFFER = 16 * 1024;   // 16KB
    
    static constexpr int MMO_GAME_RECV_BUFFER = 128 * 1024;          // 128KB
    static constexpr int MMO_GAME_SEND_BUFFER = 64 * 1024;           // 64KB
    
    static constexpr int DEFAULT_RECV_BUFFER = 64 * 1024;            // 64KB
    static constexpr int DEFAULT_SEND_BUFFER = 32 * 1024;            // 32KB
```

**버퍼 사용률 모니터링**  
운영 중인 서버의 버퍼 사용률을 모니터링하면 최적의 크기를 찾는 데 도움이 된다. 버퍼가 자주 가득 차면 크기를 늘려야 하고, 대부분 비어있다면 줄일 수 있다.

이제 Chapter 4에서 학습한 링 버퍼 기반 송수신 버퍼를 적용한 완전한 서버 코드를 구현한다.

**RingBuffer.h**

```cpp
#pragma once

#include <vector>
#include <Windows.h>

class RingBuffer {
protected:
    std::vector<BYTE> buffer_;
    int capacity_;
    int readPos_;
    int writePos_;
    
public:
    RingBuffer(int capacity)
        : buffer_(capacity)
        , capacity_(capacity)
        , readPos_(0)
        , writePos_(0)
    {}
    
    void Clear() {
        readPos_ = 0;
        writePos_ = 0;
    }
    
    int GetCapacity() const { return capacity_; }
    
    int GetDataSize() const {
        if (writePos_ >= readPos_) {
            return writePos_ - readPos_;
        } else {
            return capacity_ - (readPos_ - writePos_);
        }
    }
    
    int GetFreeSize() const {
        if (writePos_ >= readPos_) {
            return capacity_ - (writePos_ - readPos_) - 1;
        } else {
            return readPos_ - writePos_ - 1;
        }
    }
    
    int GetContinuousDataSize() const {
        if (writePos_ >= readPos_) {
            return writePos_ - readPos_;
        } else {
            return capacity_ - readPos_;
        }
    }
    
    int GetContinuousFreeSize() const {
        if (writePos_ >= readPos_) {
            int freeSize = capacity_ - writePos_;
            if (readPos_ == 0) {
                freeSize -= 1;
            }
            return freeSize;
        } else {
            return readPos_ - writePos_ - 1;
        }
    }
    
    BYTE* GetReadPtr() { return &buffer_[readPos_]; }
    BYTE* GetWritePtr() { return &buffer_[writePos_]; }
    
    void MoveReadPos(int size) {
        readPos_ = (readPos_ + size) % capacity_;
    }
    
    void MoveWritePos(int size) {
        writePos_ = (writePos_ + size) % capacity_;
    }
};
```

**RecvBuffer.h**

```cpp
#pragma once

#include "RingBuffer.h"
#include <WinSock2.h>

class RecvBuffer : public RingBuffer {
public:
    RecvBuffer(int capacity) : RingBuffer(capacity) {}
    
    // WSARecv를 위한 버퍼 준비
    bool GetRecvBuffer(WSABUF& wsaBuf) {
        int freeSize = GetContinuousFreeSize();
        if (freeSize == 0) {
            return false;
        }
        
        wsaBuf.buf = (char*)GetWritePtr();
        wsaBuf.len = freeSize;
        return true;
    }
    
    // 수신 완료 처리
    bool OnRecvCompleted(int bytesReceived) {
        if (GetFreeSize() < bytesReceived) {
            return false;
        }
        
        MoveWritePos(bytesReceived);
        return true;
    }
    
    // 데이터 읽기 (Peek - 읽기 포인터 이동 안 함)
    bool Peek(BYTE* dest, int size) {
        if (GetDataSize() < size) {
            return false;
        }
        
        int firstSize = std::min(size, capacity_ - readPos_);
        memcpy(dest, &buffer_[readPos_], firstSize);
        
        if (size > firstSize) {
            memcpy(dest + firstSize, &buffer_[0], size - firstSize);
        }
        
        return true;
    }
    
    // 데이터 소비
    void Consume(int size) {
        MoveReadPos(size);
    }
};
```

**SendBuffer.h**

```cpp
#pragma once

#include "RingBuffer.h"
#include <WinSock2.h>
#include <mutex>

class SendBuffer : public RingBuffer {
private:
    std::mutex mutex_;
    
public:
    SendBuffer(int capacity) : RingBuffer(capacity) {}
    
    // 데이터 쓰기
    bool Write(const BYTE* data, int size) {
        std::lock_guard<std::mutex> lock(mutex_);
        
        if (GetFreeSize() < size) {
            return false;
        }
        
        int firstSize = std::min(size, capacity_ - writePos_);
        memcpy(&buffer_[writePos_], data, firstSize);
        
        if (size > firstSize) {
            memcpy(&buffer_[0], data + firstSize, size - firstSize);
        }
        
        MoveWritePos(size);
        return true;
    }
    
    // WSASend를 위한 버퍼 준비
    int PrepareSendBuffers(WSABUF wsaBufs[2]) {
        std::lock_guard<std::mutex> lock(mutex_);
        
        int dataSize = GetDataSize();
        if (dataSize == 0) {
            return 0;
        }
        
        // 첫 번째 버퍼
        int firstSize = GetContinuousDataSize();
        wsaBufs[0].buf = (char*)GetReadPtr();
        wsaBufs[0].len = firstSize;
        
        // 두 번째 버퍼 (링 버퍼가 배열 끝을 넘어가는 경우)
        if (dataSize > firstSize) {
            wsaBufs[1].buf = (char*)&buffer_[0];
            wsaBufs[1].len = dataSize - firstSize;
            return 2;
        }
        
        return 1;
    }
    
    // 송신 완료 처리
    void OnSendCompleted(int bytesSent) {
        std::lock_guard<std::mutex> lock(mutex_);
        MoveReadPos(bytesSent);
    }
    
    // 송신 대기 중인 데이터 크기
    int GetPendingSize() {
        std::lock_guard<std::mutex> lock(mutex_);
        return GetDataSize();
    }
};
```

```cpp
#include "Session.h"
#include "ServerGlobal.h"
#include <iostream>

Session::Session()
    : socket_(INVALID_SOCKET)
    , sessionId_(0)
    , state_(SessionState::Idle)

    , recvBuffer_(BufferConfig::DEFAULT_RECV_BUFFER)

    , sendRegistered_(false)

    recvOverlapped_.owner = this;
}

Session::~Session() {
    if (socket_ != INVALID_SOCKET) {
        closesocket(socket_);
    }
}

void Session::Reset(SOCKET socket, uint64_t sessionId) {
    socket_ = socket;
    sessionId_ = sessionId;
    state_ = SessionState::Idle;
    refCount_ = 1;
    sendRegistered_ = false;
    
    recvBuffer_.Clear();
    sendBuffer_.Clear();
    
    ZeroMemory(&recvOverlapped_.overlapped, sizeof(OVERLAPPED));
    
    UpdateRecvTime();
}

void Session::Initialize() {
    CreateIoCompletionPort((HANDLE)socket_, g_iocpHandle,
                           (ULONG_PTR)this, 0);
    
    state_ = SessionState::Connected;
    PostRecv();
    
    std::cout << "[Session] 세션 " << sessionId_ << " 연결됨" << std::endl;
}

void Session::Disconnect() {
    SessionState expected = SessionState::Connected;
    
    if (!state_.compare_exchange_strong(expected, SessionState::Disconnecting)) {
        return;
    }
    
    std::cout << "[Session] 세션 " << sessionId_ << " 연결 해제" << std::endl;
    
    if (socket_ != INVALID_SOCKET) {
        closesocket(socket_);
        socket_ = INVALID_SOCKET;
    }
    
    state_ = SessionState::Disconnected;
    g_sessionPool->Release(this);
    Release();
}

void Session::PostRecv() {
    if (state_ != SessionState::Connected)
        return;
    
    ZeroMemory(&recvOverlapped_.overlapped, sizeof(OVERLAPPED));
    
    WSABUF wsaBuf;
    if (!recvBuffer_.GetRecvBuffer(wsaBuf)) {
        std::cout << "[Session] 수신 버퍼 부족" << std::endl;
        Disconnect();
        return;
    }
    
    DWORD flags = 0;
    DWORD bytesReceived = 0;
    
    AddRef();
    
    int result = WSARecv(socket_, &wsaBuf, 1, &bytesReceived,
                         &flags, &recvOverlapped_.overlapped, nullptr);
    
    if (result == SOCKET_ERROR) {
        int error = WSAGetLastError();
        if (error != WSA_IO_PENDING) {
            std::cout << "[Session] WSARecv 실패: " << error << std::endl;
            Release();
            Disconnect();
        }
    }
}

void Session::Send(const BYTE* data, int size) {
    if (state_ != SessionState::Connected)
        return;
    
    if (!sendBuffer_.Write(data, size)) {
        std::cout << "[Session] 송신 버퍼 부족" << std::endl;
        Disconnect();
        return;
    }
    
    // 이미 송신 중이면 등록만 하고 리턴
    if (sendRegistered_.exchange(true) == true) {
        return;
    }
    
    RegisterSend();
}

void Session::RegisterSend() {
    WSABUF wsaBufs[2];
    int bufferCount = sendBuffer_.PrepareSendBuffers(wsaBufs);
    
    if (bufferCount == 0) {
        sendRegistered_ = false;
        return;
    }
    
    SendOverlapped* sendOv = new SendOverlapped;
    ZeroMemory(&sendOv->overlapped, sizeof(OVERLAPPED));
    sendOv->owner = this;
    
    AddRef();
    
    DWORD bytesSent = 0;
    int result = WSASend(socket_, wsaBufs, bufferCount,
                        &bytesSent, 0,
                        &sendOv->overlapped, nullptr);
    
    if (result == SOCKET_ERROR) {
        int error = WSAGetLastError();
        if (error != WSA_IO_PENDING) {
            std::cout << "[Session] WSASend 실패: " << error << std::endl;
            Release();
            delete sendOv;
            sendRegistered_ = false;
            Disconnect();
        }
    }
}

void Session::ProcessRecv(DWORD bytesTransferred) {
    UpdateRecvTime();
    
    if (bytesTransferred == 0) {
        Disconnect();
        return;
    }
    
    if (!recvBuffer_.OnRecvCompleted(bytesTransferred)) {
        std::cout << "[Session] 수신 버퍼 오버플로우" << std::endl;
        Disconnect();
        return;
    }
    
    // 패킷 파싱
    while (true) {
        // 헤더를 읽을 수 있는지 확인
        if (recvBuffer_.GetDataSize() < sizeof(PacketHeader)) {
            break;
        }
        
        // 헤더 읽기
        PacketHeader header;
        if (recvBuffer_.GetContinuousDataSize() >= sizeof(PacketHeader)) {
            header = *(PacketHeader*)recvBuffer_.GetReadPtr();
        } else {
            recvBuffer_.Peek((BYTE*)&header, sizeof(PacketHeader));
        }
        
        // 패킷 크기 검증
        if (header.size < sizeof(PacketHeader) || 
            header.size > MAX_PACKET_SIZE) {
            std::cout << "[Session] 잘못된 패킷 크기: " 
                     << header.size << std::endl;
            Disconnect();
            return;
        }
        
        // 패킷 전체가 도착했는지 확인
        if (recvBuffer_.GetDataSize() < header.size) {
            break;
        }
        
        // 패킷 처리
        OnPacket(header);
        
        // 버퍼에서 패킷 제거
        recvBuffer_.Consume(header.size);
    }
    
    PostRecv();
}

void Session::ProcessSend(SendOverlapped* sendOv, DWORD bytesTransferred) {
    delete sendOv;
    
    sendBuffer_.OnSendCompleted(bytesTransferred);
    
    // 송신 버퍼에 데이터가 남아있으면 계속 전송
    if (sendBuffer_.GetPendingSize() > 0) {
        RegisterSend();
    } else {
        sendRegistered_ = false;
    }
}

void Session::OnPacket(const PacketHeader& header) {
    // Echo 처리 (받은 패킷을 그대로 전송)
    std::vector<BYTE> packetData(header.size);
    recvBuffer_.Peek(packetData.data(), header.size);
    
    Send(packetData.data(), header.size);
}

void Session::AddRef() {
    refCount_.fetch_add(1);
}

void Session::Release() {
    int32_t refCount = refCount_.fetch_sub(1) - 1;
    if (refCount == 0) {
        // 세션 풀이 관리
    }
}

void Session::UpdateRecvTime() {
    lastRecvTime_ = std::chrono::steady_clock::now();
}

bool Session::IsTimeout(int32_t timeoutSeconds) const {
    auto now = std::chrono::steady_clock::now();
    auto elapsed = std::chrono::duration_cast<std::chrono::seconds>(
        now - lastRecvTime_).count();
    return elapsed >= timeoutSeconds;
}
```
