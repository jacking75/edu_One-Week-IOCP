### Day 1: IOCP의 기초와 첫 번째 서버

**Chapter 1. IOCP 게임 서버 프로그래밍 시작하기**

이 챕터에서는 IOCP(Input/Output Completion Port)가 무엇인지, 왜 게임 서버에서 IOCP를 사용해야 하는지에 대한 근본적인 이해를 다룹니다. 기존의 Select, WSAAsyncSelect, WSAEventSelect 모델과 비교하여 IOCP의 장점을 명확히 이해하고, IOCP의 동작 원리를 그림과 함께 학습합니다.

- 1.2 IOCP란 무엇인가?
- 1.3 IOCP의 동작 원리와 아키텍처
- 1.4 개발 환경 설정 (Visual Studio 2022, Windows 11)
- 1.5 프로젝트 구성과 기본 설정

- 2.1 IOCP 핵심 API 이해하기

### Day 2: 연결 관리와 세션 처리

**Chapter 3. 클라이언트 연결 관리**

실제 게임 서버에서는 수백, 수천 명의 동시 접속자를 관리해야 합니다. 이 챕터에서는 클라이언트의 접속, 연결 유지, 연결 해제를 안전하게 처리하는 방법을 다룹니다. AcceptEx를 이용한 비동기 Accept와 세션 객체 설계 패턴을 학습합니다.

- 3.1 동기 Accept vs 비동기 Accept
- 3.2 AcceptEx를 이용한 고성능 Accept
- 3.3 세션(Session) 객체 설계
- 3.4 세션 풀(Pool) 관리 기법
- 3.5 연결 해제 처리 (Graceful Shutdown)
- 3.6 **전체 서버 코드**

**Chapter 4. 송수신 버퍼 관리**

네트워크 게임에서 가장 중요한 부분 중 하나는 효율적인 버퍼 관리입니다. 링 버퍼 구조를 구현하고, Partial Send/Recv 문제를 해결하며, 메모리 할당 최적화 기법을 다룹니다.

- 4.1 링 버퍼(Ring Buffer) 구조 이해
- 4.2 송신 버퍼 구현 (Send Buffer)
- 4.3 수신 버퍼 구현 (Recv Buffer)
- 4.4 Partial Send/Recv 처리
- 4.5 버퍼 크기 최적화 전략
- 4.6 **전체 서버 코드**

- 5.5 패킷 버퍼 클래스 구현

- 7.1 IOCP 멀티스레드 환경 이해
- 7.2 경쟁 조건(Race Condition)과 데드락
- 7.3 Critical Section과 Mutex
- 7.4 InterlockedXXX 함수군 활용
- 7.5 세션별 작업 직렬화 기법
- 7.6 참조 카운팅(Reference Counting)을 통한 안전한 세션 관리
- 7.7 **전체 서버 코드**
